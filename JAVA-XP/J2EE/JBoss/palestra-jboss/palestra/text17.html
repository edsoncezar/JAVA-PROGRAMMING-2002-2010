Arquitetura interna do JBoss

  • O Microkernel JMX (Java Management Extensions) é o que permite ao JBoss ser completamente implementado utilizando plug-ins baseados em componentes

  • JMX é uma ferramenta para integração de software. Ele provê um canal comum para se integrar módulos, contêineres e plug-ins

  • Componentes são declarados como serviços MBean que são carregados pelo JBoss. Estes componentes podem ser logo após, administrados utilizando-se o JMX
