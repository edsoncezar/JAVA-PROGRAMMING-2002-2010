No capítulo anterior, fomos apresentados aos conceitos de programação orientada a objetos. Esses conceitos agora serão apresentados de uma forma prática, aplicados à linguagem Java. Programas escritos em Java são orientados a objetos desde o início, pois cada programa, é uma classe. Um típico programa em Java consiste de um ou mais arquivos de texto, cada qual contendo pelo menos uma declaração que define uma classe. Todos os métodos e campos de dados estão necessariamente definidos dentro uma determinada classe e a ela pertencem. Não há variáveis globais ou funções independentes como em C ou C++.

Nos três capítulos seguintes iremos introduzir os conceitos básicos de Java, como sua estrutura léxica, tipos de dados, expressões e outras características comuns à qualquer linguagem de programação, sem detalhar muito a sua implementação da POO. Depois estudaremos mais a fundo a criação e o uso de classes, objetos e métodos em Java. Como impossível estudar Java sem levar em conta a sua natureza orientada a objetos (todo programa Java começa com uma declaração de classe), será inevitável depararmos com exemplos de criação de objetos, utilização de métodos, etc. Estes tópicos serão explicados aqui de forma superficial, mas merecerão uma abordagem mais detalhada em capítulos posteriores.

A melhor forma de introduzir os conceitos de uma linguagem com exemplos reais. É o que faremos neste capítulo e no próximo. Você pode testar todos os programas usados nos exemplos no seu próprio computador. Para isto é preciso que tenha no mínimo uma das configurações seguintes:

  • O Java Delopment Kit (JDK 1.02 ou superior) e um editor de textos (Notepad, Vi, WinEdit, etc.); ou
  • Um Ambiente Integrado de Desenvolvimento (IDE) para a linguagem Java como o Microsoft Visual J++, Sun Java WorkShop, Symantec Café, Borland Java ou outro.

Se você está usando um IDE, deve seguir a documentação que o acompanha para saber como compilar e rodar programas em Java. Como o JDK é gratuito, e pode ser descarregado pela Internet, as instruções de compilação e execução mostradas aqui serão referentes a ele. Veja como conseguir o JDK da Sun e documentação sobre Java no apêndice B. Lá também voc encontrará informações de como instalá-lo e configurá-lo. Se você pretende acompanhar os exemplos deste capítulo, instale e configure o JDK agora.


Exemplos de Programas Simples em Java

 

A seguir, mostraremos vários programas em Java. Se voc quiser digitá-los e testá-los em seu computador (altamente recomendável), tenha o cuidado de digitar o código exatamente da forma como é apresentado. Letras minúsculas e maiúsculas são diferentes em Java. Se na listagem aparece "String" e você digita "string", com "s" minúsculo, o compilador acusará um erro e o seu programa não ser compilado.

Esta regra também vale para o nome do arquivo. Mesmo que o Windows95 não faça distinção entre maiúsculas e minúsculas, quem efetivamente executa o programa compilado é o interpretador Java (seja via browser ou linha de comando) que considera maiúsculas diferentes de minúsculas.

O seguinte programa é o clássico "Hello World" escrito como uma aplicação Java. Depois de compilado e executado ele imprime na tela (linha de comando) as palavras "Saudações do mundo de Java!".

class HelloWorld {



    public static void main (String[] args) {



        System.out.println("Saudações do mundo de Java!");

    }

}

Digite, compile e rode o programa acima no seu computador (os detalhes dependem do ambiente de desenvolvimento que voc estiver usando). Se você estiver usando o JDK, faça o seguinte:

  1. Crie um diretório para armazenar seus programas Java e salve o programa acima em um arquivo "hello.java", nesse diretório;
  2. Abra uma janela de comando (MS-DOS Prompt) ou shell e mude para o diretório onde está o programa.
  3. Para compilar, rode o compilador Java (javac) seguido do nome do seu programa:

    javac hello.java

    Se não houver erros, o programa foi compilado e o prompt reaparecerá (se houver erros, volte e corrija-os). Se você listar o conteúdo do seu diretório agora, verá que o compilador criou um arquivo chamado "HelloWorld.class". Observe que este é o nome que demos para a classe no programa acima.

  4. Finalmente, para rodar o seu programa, invoque o interpretador Java seguido do nome da classe:

    java HelloWorld

    O programa, então, deve exibir na tela o texto seguinte:

    Saudações do mundo de Java!

Como funciona? O programa consiste de uma declaração de classe (que representa o programa-objeto) e de um método especial (main()) que contém as instruções da principal linha de execução do programa (onde ele começa). Neste caso, há apenas uma instrução no main():

System.out.println("Saudações do mundo de Java!");

Não vamos entrar em detalhes sobre esta sintaxe. No momento só precisamos saber que essa instrução imprime o conteúdo de uma cadeia de caracteres que é passada entre parênteses. Outro detalhe importante é o ponto-e-vírgula (;). Assim como várias outras linguagens de programação, em Java, toda instrução simples deve terminar em ponto-e-vírgula.

A palavra reservada "class" precede o nome da classe que chamamos de HelloWorld. Todo o código do programa fica dentro desta classe, delimitado pelas chaves { e }. Dentro da classe, está o método que também envolve com chaves toda a sua implementação.

O nome do método é apenas main, mas a sua declaração traz outras palavras: public static e void. É um método especial e deve estar presente em todo programa standalone (que não seja applet). Quando o interpretador Java roda o programa, o primeiro método que ele chama é o HelloWorld.main(). O método main deve ser sempre declarado da mesma forma. Se for usada uma sintaxe diferente, o interpretador não o encontrará.

Vamos mostrar mais dois exemplos que você pode compilar e rodar. Não detalharemos seu funcionamento aqui pois tudo será explicado mais adiante, mas é interessante que você os digite e compile para ir se familiarizando com o seu ambiente de desenvolvimento.

Esta outra aplicação faz o mesmo que a anterior mas imprime as palavras "Saudações Javanesas!" em uma janela aberta na tela. É uma aplicação escrita para usar a interface gráfica (GUI). Tudo está dentro da declaração de classe HelloWorld2, exceto uma declaração "import" que importa a biblioteca onde estão as classes básicas do sistema de janelas.


import java.awt.*;



class HelloWorld2 extends Frame {



    HelloWorld2() {



        super("Applet HelloWorld2");

        Label hello = new Label("Saudações Javanesas!");

        add("Center",hello);

        resize(200, 100);

        show();

    }



    public static void main (String args[]) {



        new HelloWorld2();

    }

}

Detalhes sobre a implementação do programa acima serão vistos no decorrer do curso, nos capítulos sobre Applets e sobre programação da interface gráfica. Observe que este programa tem um outro método, além de main, chamado HelloWorld2() (o mesmo nome que a sua classe). Esse método especial chama-se método construtor e é utilizado para inicializar objetos quando são criados. E é só isso que o main faz aqui: cria um objeto.

Se você rodar o programa irá ver uma janela do Windows com o texto "Saudações Javanesas!" dentro dela, como na figura abaixo:

Java chama esse tipo de janela de "Frame" e essa aplicação, como qualquer outra aplicação de janelas, é um "Frame". Se ela "é", é porque estende Frame como uma subclasse. Isto está representado na declaração da classe HelloWorld2 através da palavra reservada "extends".

Uma terceira versão do "Hello World" é um programa que roda como uma applet, dentro de uma página Web:

import java.awt.*;



import java.applet.*;



public class HelloWorld3 extends Applet  {



    public void paint(Graphics g) {



        g.drawString("Saudações Javanesas!", 20, 20);



    }



}

Se você tentar rodar este programa a partir da linha de comando, o interpretador acusará um erro, pois não encontrará o método main. Esta versão não contém método main porque uma applet e só pode ter a sua execução iniciada por um browser. A applet contém métodos especiais que regulam o seu ciclo de vida, eventos, apresentação, etc. como veremos no capítulo sobre applets mais adiante. Um deles, chamado paint(), é invocado automaticamente pelo browser e fornece um objeto gráfico através do qual se pode desenhar linhas, definir cores e fontes, e imprimir texto como no programa acima.

Assim como o programa anterior era um "Frame", este é um "Applet". Toda applet é uma subclasse de Applet. "Applet" e "Frame" são classes que fazem parte da biblioteca de classes da linguagem Java, chamada de API. A classe Applet, está organizada abaixo de uma biblioteca chamada de "java.applet ". Frame e Label são classes que pertencem à biblioteca "java.awt". Usa-se então "import" no início do programa para informar a localização das classes utilizadas.

Como vimos, o programa acima não roda sozinho. Applets não são aplicações independentes; precisam de um outro programa que os inicie. Para utilizar a applet necessário "chamá-la" através de uma página Web usando um descritor HTML especial como mostrado a seguir:

<!--início da página -->



<p> A seguir, uma applet...



<applet code="HelloWorld3.class" height=50 width=200>



<p>Se você está lendo isto é porque seu browser não suporta Java!



</applet>



<!-- restante da página -->

Digite o programa da applet e salve-o em um arquivo com o nome HelloWorld3.java. Neste caso, é importante que o programa-fonte (.java) tenha o mesmo nome que a classe. Depois inclua o código HTML acima em um arquivo HTML já existente ou crie um novo (use um arquivo de texto com a extensão ".html"). Você pode visualizar a applet dentro da página carregando-a em seu browser (Netscape, Internet Explorer, HotJava ou outro que suporte Java) ou isoladamente, usando o Applet Viewer, que é um programa distribuído com o JDK. Para rodá-lo digite:

appletviewer nome-da-página.html

Onde nome-da-página.html é o nome que você deu ao arquivo HTML onde está a chamada para a sua applet. A figura abaixo ilustra a applet acima rodando dentro desta página.

Exercícios

 
  1. Pratique mais com os programas apresentados neste capítulo. Modifique linhas do código e veja os erros que você obtém. É importante se familiarizar com o ambiente de desenvolvimento antes de continuar.
  2. Todas as classes que compõem a biblioteca fundamental da linguagem Java são escritas em Java. Se você instalou o JDK, pode encontrar as listagens do código-fonte de todas as classes no seu sub-diretório src (por exemplo c:\java\src ou /java/src ). Veja as listagens de java/awt/Frame.java, java/awt/Label.java e java/applet/Applet.java.
  3. Procure applets de domínio público na Internet e veja como são escritos. Tente identificar seus métodos, variáveis e classes. Procure em http://www.gamelan.com e em http://java.sun.com.
  4. Transfira o guia da API da Linguagem Java em HTML, encontrado em http://java.sun.com . Este guia contém uma referência de todas as classes, métodos e variáveis da biblioteca fundamental da linguagem Java e pode ser lido através de um browser. Instale-o e veja a referência das classes que usamos nos exemplos deste capítulo.
