GLOSSÁRIO
J2EE 2 Platform, Enterprise Edition

Este glossário define termos de acordo com seu uso no contexto da JAVA 2 Platform, Enterpresie Edition (Plataforma J2EE). Muitos dos termos possue uso mais genérico. Por exemplo, o termo handle é muito usado nas linguagens de programação modernas, mas aqui ele é definido em termos do seu uso na tecnologia Enterprise JavaBeans(EJB)

A B C D E F G H I J L M N O P Q R S T U V X Z

declaração JSP
Um elemento de scripting JSP que declara métodos e variáveis em um arquivo JSP.

delegação
Um ato pelo qual um diretor autoriza outro diretor a usar sua identidade ou privilégios com algumas restrições.

Deployer
Um pessoa que instala módulos e aplicações J2EE em um ambiente operancional.

descritor de distribuição
Um arquivo XML fornecido com cada módulo e aplicação, que descreve como eles devem ser distribuídos. O descritor de distribuição instrui uma ferramenta de de distribuição a distribuir um módulo ou aplicação como opções de contêiner específicas e descreve requisitos de configuração específicos que um Deployer precisa resolver.

destino
Um objeto administrado pelo JMS, que encapsula a identidade de uma fila ou tópico JMS, Veja "sistema de mensagem ponto a ponto", "sistema de mensagem publicar/assinar", neste glossário.

diretiva JSP
Um elemento JSP que dá uma instrução ao contêiner JSP e interpretado no momento da tradução.

diretor
A identidade atribuída a um usuário como resultado da autenticação.

diretor do servidor
O diretor do sistema operacional do servidor em execução.

diretor do sistema operacional
Um diretor nativo do sistema operacional, em que a plataforma J2EE está executando.

distribuição
O Processo pelo qual o software é instalado em um ambiente operacional.

DOM
Document Object Model. Um árvore de objetos com interfaces para atravessar a árvore e escrever uma versão XML dela, conforme definida pela especificação W3C.

domínio da política de segurança
Um escopo sobre o qual as políticas de segurança são definidas e impostas por um administrador de segurança. Um domínio da política de segurança possui uma coleção de usuários(ou diretores), utiliza protocolo(s) automaticamente bem definido(s) para autenticar usuários(ou diretores) e pode definir grupos para simplificar a definição das políticas de segurança.

domínio de tecnologia de segurança
Um escopo sobre o qual o mesmo mecanismo de segurança é usado para impor uma política de segurança. Vários domínios de política de segurança podem existir dentro de um único domínio de tecnologia.

DTD
Document Type Definition. Uma descrição da estrutura e das propriedades de uma classe de arquivos XML. Veja XML, neste glossário.
