GLOSSÁRIO
J2EE 2 Platform, Enterprise Edition

Este glossário define termos de acordo com seu uso no contexto da JAVA 2 Platform, Enterpresie Edition (Plataforma J2EE). Muitos dos termos possue uso mais genérico. Por exemplo, o termo handle é muito usado nas linguagens de programação modernas, mas aqui ele é definido em termos do seu uso na tecnologia Enterprise JavaBeans(EJB)

A B C D E F G H I J L M N O P Q R S T U V X Z

objeto adminitrado por JMS
Um objeto JMS pré-configurado (uma fábrica de conexão do gerenciador de recursos ou um destino) criado por um administrador para o uso de clientes JMS e colocado em um namespace da JNDI

objeto EJB
Um objeto cuja classe implementa a interface remota do enterprise bean. Um cliente nunca referencia diretamente uma instância de enterprise bean; um cliente sempre referencia um objeto EJB. A classe de um objeto EJB é gerada pelas ferramentas de distribuição de um contêiner.

objeto EJB local
Um objeto que oferece as operações de ciclo de vida (criar, remover, localizar) para um enterprise bean. A classe para o objeto EJB local é gerada pelas ferramentas de distribuição do contêiner. O objeto EJB local implementa a interface local do enterprise bean. O cliente referencia um objeto EJB local para realizar operações de ciclo de vida sobre um objeto EJB. O cliente usa JNDI para localizar um objeto EJB local.

ORB
Object Request Broker. Uma biblioteca que permite a objetos CORBA localizarem e comunicarem-se um com o outro.

OTS
Object Transaction Service. Uma definição das interfaces que permitem aos objetos CORBA participarem em transações.
