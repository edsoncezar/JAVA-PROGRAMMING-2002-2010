GLOSSÁRIO
J2EE 2 Platform, Enterprise Edition

Este glossário define termos de acordo com seu uso no contexto da JAVA 2 Platform, Enterpresie Edition (Plataforma J2EE). Muitos dos termos possue uso mais genérico. Por exemplo, o termo handle é muito usado nas linguagens de programação modernas, mas aqui ele é definido em termos do seu uso na tecnologia Enterprise JavaBeans(EJB)

A B C D E F G H I J L M N O P Q R S T U V X Z

SAX
Simple API for XML. Um mecanimos de acesso serial, controlado por evento, para acessar documentos XML.

scriptlet JSP
Um elemento de scripting JSP contendo qualquer fragmento de código que é valido na linguagem de scripting usada na página JSP. A especificação JSP descreve o que é um scriptlet válido para o caso em que o atributo de página da linguagem é "java".

servidor da Web
Software que oferece serviços para acessar a Internet, uma intranet ou uma extranet. Um servidor da Web hospeda sites da Web, oferece suporte para HTTP e outros protocolos e executa programas no servidor (como scripts CGI e servlets) que realizam certas funções. Na arquitetura J2EE, um servidor da Web oferece serviços a um contêiner da Web. Por exemplo, um contêiner da Web normalmente conta com um servidor da Web para oferecer o tratamento de mensagens HTTP. A arquitetura J2EE considera que um contêiner da Web é hospedado por um servidor da Web do mesmo fornecedor, e por isso não especifica o contrato entre essas duas entidades. Um servidor da Web pode hospedar um ou mais contêineres da Web.

servidor EJB
O software fornece serviços a um contêiner EJB. Por exemplo, um contêiner EJB normalmente se baseia em um gerenciador de transação que faz parte do servidor EJB para realizar o commit em duas fases por todos os gerenciadores de recursos participantes. A arquitetura J2EE considera que um contêiner EJB é hospedado por um servidor EJB do mesmo fornecedor e por isso não especifica o contrato entre essas duas entidades. Um servidor EJB pode hospedar um ou mais contêineres EJB.

servidor J2EE
A parte de runtime de um produto J2EE. Um servidor J2EE oferece EJB e/ou contêineres da Web.

servlet
Um programa Java que estende a funcionalidade de um servidor da Web, gerando conteúdo dinâmico e interagindo com os clientes da Web usando um paradigma pedido-resposta.

sessão
Um objeto usado por um servlet para rastrear a interação de um usuário com uma aplicação da Web por vários pedidos HTTP.

sessão JMS
Um contexto de único thread para enviar e receber mensagens JMS. Um sessão JMS pode não ter transação, ter transação local ou participar em uma transação distribuída.

sistema de informações da empresa
As aplicações que compreendem o sistema existente de uma empresa para o tratamento de informações de toda a empresa. Essas aplicações oferecem uma infra-estrutura de informação para uma empresa. O sistema de informações de uma empresa oferece um conjunto bem definido de serviços aos seus clientes. Esses serviços são expostos aos clientes como interfaces locais e/ou remotas. Alguns exemplos de sistemas de informação da empresa são: sistemas de planejamento de recursos da empresa, sistemas de processamento de transação no mainframe e sistemas de banco de dados legados.

sistema de mensagem ponto a ponto
Um sistema de mensagem montado em torno do conceito de filas de mensagem. Cada mensagem é endereçada para uma fila específica; os clientes extraem mensagens da(s) fila(s) estabelecida(s) para conter suas mensagens.

sistema de mensagem publicar/assinar
Um sistema de mensagem em que os clientes endereçam mensagens a um nó específico em uma hierarquia de conteúdo. Os publicadores e assinantes geralmente são anônimos e podem publicar ou assinar dinamicamente para a hierarquia de conteúdo. O sistema cuida da distribuição das mensagens que chegam a partir dos vários publicadores de um nó para os seus vários assinantes.

SQL
Structured Query Language. A linguagem de banco de dados relacional padronizada para a definição de objetos de banco de dados e a manipulação de dados.

SQL/J
Um conjunto de padrões que inclui especificações para incorporar instruções SQL em métodos na linguagem de programação Java e especificações para a chamada de métodos java estáticos como procedimentos armazenados SQL e funções definidas pelo usuário. Um verificador SQL pode ser usado para detectar erros em instruções SQL estáticas em tempo de desenvolvimento de programa, em vez de, por exemplo, com um driver JDBC em tempo de execução.

SSL
Secure Socket Layer. Um protocolo de segurança que oferece privacidade pela Internet. O protocolo permite que aplicações cliente-servidor se comuniquem de uma forma que não pode ser bisbilhotada ou adulterada. os sevidores sempre são autenticados e os clientes são opcionalmente autenticados.
