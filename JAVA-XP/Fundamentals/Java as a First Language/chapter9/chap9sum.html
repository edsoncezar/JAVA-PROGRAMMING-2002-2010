Summary of Chapter 9

  • Java does not supply numeric input facilities, the ValidatedInput class from this chapter is intended to be (re)used for this requirement.

  • The FormattedOutput class from this chapter intended to be (re)used for precise control over the output of numeric values.

  • Complex String processing is essential to many applications.

  • Exceptions can be divided into Runtime and non-Runtime classes.

  • When a non-Runtime exception is thrown it must be caught within the action or stated in a throws clause in the action's header.

  • Clients using an action whose header indicates that it might throw a non-Runtime exception must catch the exception.

  • A Runtime exception need not be handled, nor need it be mentioned in a throws clause, but if it is not caught it may propagate and halt the program.

  • The reasons why an exception is thrown may be communicated using the type of exception thrown or by constructing an exception class which encapsulates a reason.
