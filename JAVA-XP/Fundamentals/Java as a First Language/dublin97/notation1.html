Design notations - Introduction

This part of the tutorial introduces the design notation which will be used. These include augmented JSP schematics to describe the implementation of actions and UML style notation for the instance and state transition diagrams.

UML notation is not used for the class diagrams as the notation is not particularly graphically expressive and so is not considered as suitable as the alternative Booch/ OML style. Conversion between the two styles is not particularly complex and could be introduced at a later stage.

Details of the UML notation can be found at http:\\www.rational.com.
