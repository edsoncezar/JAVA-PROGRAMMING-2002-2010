The LimitedCounterDemonstration class

The LimitedCounterDemonstration class will have to demonstrate both of its constructors. In order to demonstrate their effectiveness the inherited numberCountedIs() action will have to be used.

All four enquiry actions will also have to be demonstrated. The boolean actions will have to be demonstrated twice; once with a true result and once with a false result. In order to accomplish this the count() (or unCount()) action will have to be used to increment the counter from its minimum to maximum values. (To facilitate this is the alternative constructor should be used to establish a very small range e.g. minimum 10 and maximum 12.)

Will the reset() action publicly inherited by LimitedCounter from BasicCounter need to be demonstrated? Will it be effective?
