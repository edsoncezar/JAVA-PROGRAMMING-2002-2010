Demonstrating the BasicCounter class

To follow the design, build, test … technique the BasicCounter class should be shown to be operating correctly at this stage, before proceeding to extend it into the LimitedCounter class. To do this it will have to be temporarily re-implemented as a non-abstract public class with public actions. Having done this a demonstration harness client can be designed and constructed which will allow all the actions to be shown to be apparently operating correctly.

The class declaration should be changed from: 

0010   abstract class BasicCounter {

to: 


0010 public class BasicCounter {

and the non-public actions as follows: 


0024   protected void count() {

to: 


0024   public void count() {

It is probably too early to introduce formal black box testing, but the point that demonstrating is not really sufficient should be made at this stage.
