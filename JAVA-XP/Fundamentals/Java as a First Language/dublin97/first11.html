The LimitedCounter constructors

 

The implementation of this design as far as the end of the constructors is as follows. 

0008 package Counters;
0009 
0010 abstract class LimitedCounter extends BasicCounter { 
0011 
0012 static private final int DEFAULT_MINIMUM = 0;
0013 static private final int DEFAULT_MAXIMUM = 999;
0014 
0015 private int miniumumCount;  
0016 private int maximumCount;  
0017 
0018 
0019    protected LimitedCounter(){
0020       this( DEFAULT_MINIMUM, DEFAULT_MAXIMUM);
0021    }  // End Default Constructor.
0022    
0023    protected LimitedCounter( int minToCount, int maxToCount){ 
0024        super( minToCount);
0025        miniumumCount = minToCount;
0026        maximumCount  = maxToCount;
0027    }  // End alternative Constructor.

On lines 0019 and 0021 he specification static indicates the class wide nature of the declaration, private its totally restricted visibility and final that it is a constant.

The default constructor is declared on lines 0025 to 0027 and is implemented as a indirecting call of the alternative constructor passing the two manifest values as arguments. This again reinforces the concept that actions should be implemented by making use of existing action wherever possible.

The alternative constructor commences by calling its super (BasicCounter) constructor passing minToCount as the argument. This will ensure that a newly constructed instance will be at the lower limit of its range. The alternative constructor concludes by setting the values of the two instance attributes to the value of their respective argument.
