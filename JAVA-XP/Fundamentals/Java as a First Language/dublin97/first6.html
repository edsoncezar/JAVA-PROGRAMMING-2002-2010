The setCountTo() action is implemented by setting the value of the counted attribute to the avlue of its argument setTo.

The value of this attribute (counted) is returned by the numberCountedIs() action. The last action, reset(), transfers the value of theInitialCount to counted in order to restore the initial state of the instance.

The implementation of this first class introduces the essential aspects of data encapsulation. There is no way of changing, or determining, the value of theInitialCount after construction but the value of counted can be manipulated using various logical actions. This avoids the misconception of seeing a class as just a wrapper for a single primitive variable instance.
