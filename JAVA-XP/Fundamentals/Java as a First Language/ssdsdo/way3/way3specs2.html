Waypoint 3 is a complete design, build, demonstrate and document exercise.

Specification

The specification is to implement and demonstrate a class hierarchy which is concerned with the modeling of a radio. The class hierarchy, indicating that all classes are contained within the radio package, is as follows.

  • The SimpleRadio class contains a boolean attribute determining if the radio is turned off or on.
  • The VolumeRadio class contains an int attribute determining the loudness of the sound.
  • The TuningRadio class contains a double attribute determining the FM frequency.
  • The RadioException class allows attempts to set the volume or frequency to inappropriate values to be reported.

The class diagrams of the SimpleRadio and VolumeRadio classes are as follows.

  

  • The boolean attribute onOrOff indicates the state, on or off, of the radio.
  • The constructor will ensure that the radio is switched off, by setting onOrOff to false.
  • The switchOn() and switchOff() methods set the value of onOrOff true and false respectively.
  • The isSwitchedOn() and isSwitchedOff() methods indicate the switched state of the radio.
  • The toString() method will return a string describing the state of the radio.

  • The int attribute volume indicates the required loudness of the speakers, between 0 and 10.
  • The constructor will ensure that the radio is switched off, and set volume to 0.
  • The getVolume() method will obtain the value of the volume attribute.
  • The louder() and quieter() methods will increment or decrement the value of volume, throwing a suitable exception if the resulting volume is too low or too high.
  • The mute() method will set the value of the volume to 0.
  • The isSilent() method will return true if the value of volume is 0.
  • The toString() method will return a string describing the state of the radio, this will be restricted to stating that the radio is switched off if that is the case.

The class diagram for the TuningRadio class will not be given, its description is as follows.

  • The double attribute frequency indicates the required tuning of the radio, between 88.0 and 109.0.
  • The constructor will ensure that the radio is switched off, set volume to 0 and set frequency to 93.5 (Radio 4).
  • The setFrequency() and getFrequency() methods are both public and manipulate the frequency attribute as appropriate.
  • The tuneUp() and tuneDown() methods increment or decrement the value of frequency by 0.1.
  • The toString() method represents the state of the instance, for example:

Radio Switched Off

The radio is switched off.

Radio Switched On and silent

The radio is switched on.
The speaker is muted.

Radio Switched On & not silent

The radio is switched on.
The volume is set to 4.
It is tuned to 93.5.

The SimpleRadio class and SimpleRadioDemo classes are complete and should not be changed.

The VolumeRadio class is fully implemented but contains errors, the VolumeRadioDemo class is incomplete.

The TuningRadio class and TuningRadioDemo classes are supplied in outline only.

The RadioException class is fully implemented and should not be changed.

All demonstration programs can be executed in their current state.
