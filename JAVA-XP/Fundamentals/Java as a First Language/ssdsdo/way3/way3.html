Specification

The specification is to implement and demonstrate a class hierarchy which is concerned with the modeling of a booking of an airline seat. The class hierarchy of the airseat package, is as follows.

  • The AirlineSeat class contains attributes identifying the seat, from these attributes it can be determined if it is in an exit row, if it is a window seat or if it is an aisle seat.
  • The BookedSeat class contains attributes determining if the seat is booked and, if so, if the traveller is an INFANT, CHILD, YOUTH or ADULT and also if they are a vegetarian.
  • The PricedSeat class contains an attribute indicating the city that the passenger is travelling to or from and so can determine the price of the booking.

The class diagrams are as follows.

  

  • The argument to the constructor is a String (e.g. "16d") containing the row (1 to 26) number and seat (A to J) letter. It will throw an Exception if the argument is invalid.
  • The getRowNumber() and getSeatLetter() attributes are inquiry methods on the row and seat attributes respectively.
  • The isEmergencyExit() method returns true for rows 5, 16 and 26 and false otherwise.
  • The isWindowSeat() method returns true for seats A and J and false otherwise.
  • The isAisleSeat() method returns true for seats C, D, G and H and false otherwise.
  • The toString() method will return a string describing the seat, eg "Seat 5D EE AI" showing EE for emergency exit, WI for window and AI for aisle.

  • The constructor is as above, leaving booked false, ageType UNKNOWN and vegetarian false.
  • The bookSeat() method will set booked true and vegetarian to the value of the veg argument. The age argument is the age of the passenger in years and is used to set the value of ageType to INFANT (less than 2), CHILD less than 16, YOUTH less than 26 and ADULT otherwise. This method should throw an exception if a child or infant attempts to book an exit row seat.
  • The cancelBooking() will reset the attributes to the default values.
  • The isBooked() is an inquiry method on the booked attribute.
  • The isInfant(), isChild(), isYouth() and isAdult() are all inquiry methods on the ageType attribute.
  • The isVegetarian() is an inquiry method on the vegetarian attribute.
  • The toString() method will add more information to the same line produced by the AirlineSeat toString() method. If the seat is not booked it should only add VACANT. If the seat is booked it should add I, C, Y or A for the ageType attribute and VEG for vegetarians.

The class diagram for the PricedSeat class will not be given.

  • The constructor adds an attribute city which identifies the city that the flight was toOrFrom (all flights are to or from London and prices are for single flights only). The attribute should be one of PARIS, BERLIN, PRAGUE and MOSCOW, all of which are declared as public manifest values by the class, and should throw a suitable exception otherwise.
  • The basePrice is £50 for Paris, £70 for Berlin, £80 for Prague and £100 for Moscow.
  • The agePrice is the same as the basePrice for adults, 75% of the basePrice for youths, 50% of the basePrice for children and £0 for infants.
  • The taxes are £0 for infants, £10 for children and £25 for everyone else.
  • The price is the price to be paid and is agePrice plus taxes.
  • The toString() method adds an additional line to the output produced by the BookedSeat toString() method, showing all details of the price ( basePrice, agePrice, taxes and price).

The AirlineSeat class contains a single error.

The AirlineSeatDemo class is minimal but should not be changed.

The BookedSeat and PricedSeat classes are incomplete and need to be completed.

The PricedSeatDemo class is minimal and needs to be completed.

All demonstration programs can be executed in their current state.
