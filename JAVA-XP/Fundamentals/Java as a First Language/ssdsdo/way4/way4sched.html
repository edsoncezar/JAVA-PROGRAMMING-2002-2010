Waypoint 4 Schedule

The demonstration programs from previous waypoints only produced output on the screen. The first part of this waypoint will introduce the techniques for obtaining input from the user and for formatting numeric output.

Obtaining input from ther user is one of the most difficult and complex parts of the software development process. The user will see the software artefact, and form thier opinions of it, from thier experiences of using its user interface. Therefore producing an interface which is effective, efficient and enjoyable is essential to successful software development.

This part of the waypoint should
only take about 1 week!!!

Output of formatted numeric values.

Input of integer and floating point values.

Input of Manifest Values.

Putting these together: inputting details of a radio slot.

The second part of this waypoint will make use of the BasicMenu class from Chapter 6 of the text book, the AdaptingMenu class from Chapter 7 of the text book, the input routines from the first part of this waypoint and the RadioSlot hierarchy from waypoint 2 to develop a simple application called RadioShow. This application allows the user to maintain a list of items which comprise a radio show.

To minimise risk the application will be developed in stages. That is a small part of the product is built and then demonstrated (or better tested). Once this part is shown to be working correctly an additional part is added to it and demonstrated. This continues until the whole product has been constructed. In order to do this successfully it is essential to plan the steps and stages, and it is also very useful to estimate how much time that each stage will take. While the product is being built the actual time taken should be logged in order that you can begin to start accurately estimating how much time a specification will require.
