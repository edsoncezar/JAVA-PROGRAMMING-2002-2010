The call of readManifest(), on line 0052 and 0053, supplies a prompt for the interaction with the user and will not return until the user has entered one of the colour names from the list. An example interaction of this part of the demonstration might be as follows. 

Demonstrating readManifest() action
Please input a manifest value(s) NOT in the list 
followed by a value in the list.

Please enter a colour of the rainbow fred
Please enter one of the manifest values ...
red, orange, yellow, green, blue, indigo, violet.

Please enter a colour of the rainbow ¿ 
Please enter one of the manifest values ...
red, orange, yellow, green, blue, indigo, violet.

Please enter a colour of the rainbow green
Value returned is 3
 
End of Manifest Input Demonstration

The first two inputs, fred and july, are not acceptable inputs and the advice to the user is given, as shown. On the third attempt the user pressed the <ENTER> key (shown here as ¿) and this is also not accepted. On the fourth attempt the user entered green which is an acceptable input and the readManifest() action returns its manifest value 3.
