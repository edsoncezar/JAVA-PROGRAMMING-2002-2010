The RadioShow class contains the declaration of the listTheSlots() method as well as the declaration of a main() method. As this method is not to be visible outside this class it is declared private and as only one of them will ever be required it is also declared static. 

0053    private static void listTheSlots( Vector listToShow) {

The listTheSlots() method needs to receive a Vector as an argument. In the method declaration, on line 0053, it is named as listToShow. When this method is called from main(), on line 0043, the Vector called theList, declared on line 0033 is passed.

The argument on line 0043, when a method is called, is known as the actual argument. The argument on line 0053, when the method is declared, is known as the formal argument.

It is possible for the formal and actual arguments to have the same identifier, but it is regarded as better style to give them different names. One reason for this is that it emphasises that the listTheSlots() method is complete in itself and does not depend upon any declarations made in main(). A second reason is that a method may be called from many places in a program with different formal arguments, hence having different names for the formal and actual arguments will avoid any confusion about which actual argument is intended.
