The Die Class example

This exercise will not be assessed, but it will provide additional learning experiences to help you cope with the assessments in waypoints 1 & 2

The Problem

We would like to create a Die class for a game. This should be created so that it will be useful for any games we might create in the future which require dice. Note:
One DIE
but two DICE
Answers are available on the link at the end of this page, but you will benefit much more if you try all the stages first.

Tasks:

  1. For the Die Class, first answer the following questions
    1. What value(s) will it hold? What types of values?
    2. What does an object of type Die do? What methods will it have?
    3. Do all dice have the same number of sides for all games? What is the implication of this answer?
  2. Try to draw a class diagram for the Die Class
  3. Using previous examples as a starting point, write a Java class for the Die class
  4. Write a demonstration program to illustrate the action of the Die objects *
  5. Advanced : Extend the Die class to create a new class that contains both a minimum and maximum value. The random values returned must be within this range. The minimum and maximum values can both be set on creation of an object, or if no parameters are provided, the default values should be 1 and 6 respectively. Also supply methods which will allow the user to change the minimum and maximum values.
  6. Advanced : Draw a class diagram for this new class
  7. Advanced : Create a demonstration program which fully shows the action of at least 2 objects of this class.
