The ChangingTemperature class

The ChangingTemperature class extends the Temperature class

It supplies two constructors and adds two new public state changing methods. It does not add any attributes or protected methods.

An instance of the ChangingTemperature class is an instance of the Temperature class and so inherits all of the methods and attributes supplied by the Temperature class.

The warmer() method will increase the value of the instance by 0.1oC and the cooler() method will decrease the value by 0.1oC, hence changing the temperature recorded by the instance by changing the state of the object.

Lone 0010 states that the ChangingTemperature class extends the Temperature class, exactly as shown on the class diagram.

The two constructors are implemented as follows. 

0012     public ChangingTemperature() { 
0013        this( 0.0); 
0014     } // End ChangingTemperature default constructor.
0015  
0016     public ChangingTemperature( double setTo) { 
0017        super( setTo); 
0018     } // End ChangingTemperature alternative constructor.

The default constructor calls the alternative constructor, passing the argument 0.0 in exactly the same way as the Temperature default constructor.

The alternative constructor calls the super constructor passing on its setTo argument. The super class of the ChangingTemperature class is the Temperature class and so this is a call of the Temperature alternative constructor. This will set the state of the instance by setting the state of the theTemp attribute inherited from the Temperature class. As the ChangingTemperature class does not declare any attributes there is nothing to be placed into a well defined initial state and so there are no more steps for this constructor.

The warmer() method is declared as follows.

The method is public as it is shown on the right of a class diagram and its name is warmer. It does not return any information as this is shown by the void return type. It takes no arguments and this is shown by the empty round brackets ().

A method such as this, which takes no arguments and returns no information, will (usually) change the state of the instance..

To make the Temperature instance 0.1oC warmer the method has to:

  1. Get the current temperature
  2. Add 0.1 to the current temperature
  3. Set the temperature to the increased value.

This can be accomplished as follows. 

0021     public void warmer() { 
0022  
0023     double currentTemperature = this.getTemp();
0024  
0025        currentTemperature += 0.1; 
0026        this.setTemp( currentTemperature); 
0027     } // End warmer.

On line 0023: a local variable of the warmer() method is declared. This is not an attribute of the class, it will only exist between the point where it is declared and the } which marks the end of the method, on line 0027. An attribute is declared outside any method and exists throughout the entire class.

The local variable is called currentTemperature and is of the primitive double type. It is given the value returned from the getTemp() inquiry method which is inherited from the Temperature class. This implements stage 1 above.

On line 0025: 0.1 is added to the value in currentTemperature. The += operator is add and store, it adds whatever value is on its right hand side to whatever variable is indicated on the left hand side. This implements stage 2 above.

On line 0026 the protected setTemp() method, inherited from the Temperature class is called passing as an augment the increased value in currentTemperature. This will change the state of the instance. This implements stage 3 above.

This class can call a protected method inherited from its super class as this is the meaning of the partially public visibility indicated by protected. A demonstration client is not an extension of the Temperature class and so it cannot use the method.

A skilled developer would probably not implement the warmer() method in this way. The method is very similar to the cooler() method which is implemented as follows. 

0029     public void cooler() { 
0030        this.setTemp( this.getTemp() - 0.1); 
0031     } // End cooler. 
0032  
0029  } // End ChangingTemperature.

The three steps have now been placed in a single line. The call of setTemp() specifies as an argument the value returned from getTemp() with 0.1 subtracted from it. The effect is essentially the same as the warmer() method, changing the sate of the instance by 0.1, but avoids the need to declare a local variable to hold the value for an instant while it is adjusted.
