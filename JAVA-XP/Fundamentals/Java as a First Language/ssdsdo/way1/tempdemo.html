The TemperatureDemonstration program

The Temperature class supplies objects that can be used by other classes as a part of a program, it is not a program in its own right.

For every class that is developed it is recommended that a demonstration class for it is also developed.

The purpose of a demonstration client is to give an initial indication that the class appears to be implemented correctly. It does not prove that the class is implemented correctly or even give a high degree of confidence that it is.

To demonstrate the Temperature class, the TemperatureDemonstration program will construct three Temperature instances representing three different temperatures and show that they have been constructed correctly by showing their state with toString().

This diagram, known as an instance diagram, shows that the TemperatureDemonstration program will construct and show three instances of the Temperature class called freezing (0.0oC), boiling (100.0oC) and bodyTemp (37.8oC). It also shows that the Temperature class is an extension of the Java Object class.

An instance diagram is used to show which instances of which classes are used in which relationships with each other in order to implement a particular artefact.

In order for the TemperatureDemonstration class to be executed as a program it has to supply a main() method. When the program is started execution will begin at the start of the main() method and will finish when it comes to an end.

The declaration of a main() method, which indicates the program to be executed, must be as shown on line 0012.

Lines 0014 to 0016 declare, but do not construct, the three Temperature instances shown on the instance diagram.

Lines 0018 to 0023 output the strings supplied as arguments to the System.out.println() method calls onto the terminal. When this part of the program is executed the output will always be as follows.

 

Temperature Demonstration


This demonstration will construct and display

Three different instances of the Temperature class.


Construcing ...

The next part of the program actually constructs the three instances. 

0025        freezing = new Temperature();
0026        boiling  = new Temperature( 100.0);
0027        bodyHeat = new Temperature( 37.8);

On line 0025 the default constructor (without any arguments) is used. This should produce an instance, called freezing, which has the value 0.0oC. Lines 0026 and 0027 use the alternative constructor whose argument (100.0 or 37.8) is used to initialise the value.

The final part of the main() method shows the values of the three instances.

0029 System.out.println( "\nConstructed, showing ... \n");
0030
0031 System.out.println( "Freezing is " + freezing.toString() );
0032 System.out.println( "Boiling is " + boiling.toString());
0033 System.out.println( "Body Heat is " + bodyHeat);
0034 } // End main.
0035
0036 } // End TemperatureDemonstration.

If the Temperature class is implemented correctly this should produce the following output 

Constructed, showing ... 



Freezing  is 0.0 degrees centigrade

Boiling   is 100.0 degrees centigrade

Body Heat is 37.8 degrees centigrade

On lines 0031 and 0032 the freezing and boiling toString() methods are used to obtain the state of the instances, producing "0.0 degrees centigrade" and "100.0 degrees centigrade" respectively. Line 0033 shows that this is not necessary, as when a class instance is passed to the System.out.println() method its toString() method is called automatically.

Line 0034 is the end of the main() method and line 0036 is the end of the TemperatureDemonstration class.
