UML Class Diagrams

Classes can be illustrated graphically in UML quite simply.

Look at the SimpleCounter class


public class SimpleCounter extends Object{
  private int count;
               // the constructor
  public SimpleCounter (int startValue){
      count = startValue;
  }
                // a simple action 
  public void click (){
      count++;
  }
                // a 'getter'
  public int getValue(){
       return count;
  }
}
You can easily see how the code ties in with its UML class diagram The "+" sign is used to show something as public
The "-" sign means it is private
"#" indicates that it is protected
