The PlayTime implementation

The implementation of the PlayTime class, as far as then end of its constructors, is as follows. 

0006  package playtime; 
0007  
0008  
0009  public class PlayTime extends BasicTime { 
0010  
0011     public PlayTime() { 
0012        this( 0);
0013     } // End PlayTime default constructor.
0014  
0015     public PlayTime( int tenths) { 
0016        super( tenths);
0017     } // End PlayTime constructor.

The default constructor indirects to the alternative constructor, which calls the super, BasicTime, constructor.

The timeBetween() method is implemented as follows. 

0019     public PlayTime timeBetween( PlayTime startTime) { 
0020        return new PlayTime( this.getDuration() - 
0021                             startTime.getDuration());
0022     } // End timeBetween

The header of the method, on line 0019, is derived from the class diagram; exactly as the previous headers were. The body of the method constructs and returns a new PlayTime instance. The value of the new instance is the difference between the durations of this instance and the startTime argument.

For example if this instance has the value 1 min 12.4 seconds its duration attribute will have the value 724 ( 1*600 + 12*10 +4). Is startTime has the value 1 min 03.2 seconds its duration will be 632 ( 1*600 + 3*10 +2). The difference between these two values is 92 (tenths of a second) which is 0 min 09.2 seconds.

The implementation of the addTime() method is as follows. 

0024     public PlayTime addTime( PlayTime addTo) { 
0025        return new PlayTime( this.getDuration() + 
0026                             addTo.getDuration());
0027     } // End addTime   
0028  
0029  } // End class PlayTime

The implementation of this method is similar to that of the timeBetween() method, returning a new PlayTime instance which has a value (in tenths of a second) equal to that of the added durations of this and the addTo instances.
