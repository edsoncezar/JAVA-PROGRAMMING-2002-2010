As with the MeterReading classes, these two classes are in the gasbill package and the two source code files have to be stored in a subdirectory called gasbill. Compilation and execution of the demonstration has to be performed from the gasbill's parent directory.

Extend the GasBillDemonstration program to construct a third GasBill instance called otherBill with a made up address and reading value and show that the attributes have the correct values.

Investigate the throwing of an exception in more detail. After constructing a GasBill instance call the setNewReading() method with newReadings which are 1 below, equal to and 1 above the original reading value. Predict in advance, which of these should throw an exception and extend the demonstration program to see if your predictions are correct.

Does this activity increase your confidence in the correct implementation of the GasBill class.

Extend the VehicleOdometer class to produce a TaxiJourney class. The TaxiJourney class should also be in the taxi package of classes and adds an integer instance attribute called milesAtEnd. The class should also declare a class wide instance constant called PRICE_PER_MILE with the value 0.40.

The class should provide inquiry methods to determine

  • the value of milesAtEnd,
  • the distance travelled (getDistanceTravelled() :
    ie the difference between the inherited     milesTravelled and milesAtEnd attributes)
  • the cost of the journey ( getCostOfJourney() :
    ie     milesTravelled times PRICE_PER_MILE)
  • the value of PRICE_PER_MILE

The class should also supply a toString() method.

To attempt this exercise you should first draw a class diagram and indicate clearly what output the toString() method should produce. The class can then be implemented by carefully following the design and the implementation demonstrated with a demonstration program.

There is usually a minimum charge for a taxi journey. Adapt the costOfJourney() method to impose a minimum change of £2.50. This can be accomplished by defining a class wide constant attribute called MINIMUM_COST and using a local variable within the costOfJourney() method called localCost which is declared an initialised as follows.

double localCost = this. getDistanceTravelled() * PRICE_PER_MILE

The body of the method can be implemented as a two way if else if structure which decides if the value of localCost is less than the value of MINIMUM_COST and returns the value of MINIMUM_COST if so. If not it should return the value of localCost.

When you have done this adapt the demonstration program so that one journey is less than the MINIMUM_COST and one is more than it.

Does the TaxiJourney toString() method need to be adapted because of the MINIMUM_COST rule? If so adapt it as appropriate. When this is done paste the output of the demonstration program, including the toString() output for both instances, into your work book.
