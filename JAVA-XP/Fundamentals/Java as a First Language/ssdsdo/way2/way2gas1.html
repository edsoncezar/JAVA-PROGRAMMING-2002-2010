The MeterReading class diagram.

A MeterReading instance has two private attributes, the address of the property and the value of the reading of the meter. (For example "10 Downing Street" & 18203 or "34A Smith Square" & 92378). The class diagram is as follows.

The top of the class diagram shows that the MeterReading class extends the Object class. It also indicates that the MeterReading class is a member of the gasbill package of classes.

When a realistically sized artefact is being constructed there may be hundreds of different classes. In order to assist with managing the library of classes they are divided into collections of classes, known as packages. (Most Java developers argue that every single class should a member of a package.) When a class is declared as a member of a package it must be stored in a certain subdirectory and compiled in a certain way. This will be explained when the MeterReadingDemonstration program is described.

The state of a MeterReading instance is the combined value of both of its attributes and these can only be set, as the instance is constructed, by the arguments of the constructor.

The getAddress() and getReading() inquiry methods can be used to obtain the values of these two attributes and the toString() method will produce an String representation of the form "10 Downing Street reading 18203".
