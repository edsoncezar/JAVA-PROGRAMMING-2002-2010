The PlayTime and RadioSlot hierarchies

A RadioSlot represents a thing to be broadcast on a radio show, for example: 

Advert Sid's Used Car Lot
Start    4 min 0.0 seconds
Duration 0 min 30.0 seconds
End      4 min 30.0 seconds

Which indicates that an advert for Sid's User Car Lot will start at 4 min 0.0 seconds will have a duration of 0 min 30.0 seconds and so end at 4 min 30.0 seconds.

There are apparently five attributes:

  • The type of the slot, slotType, - Music, Advert, News or Weather.
  • A description of the slot, slotDescription, - for example "Sid's User Car Lot"
  • The startTime of the slot, - 4 min 0.0 seconds.
  • The duration of the slot - 0 min 30.0 seconds.
  • The endTime of the slot - 4 min 30.0 seconds.

The RadioSlot class diagram is as follows:

It shows that the class is in the radioslot package and has only four attributes as if the startTime and duration are known, the endTime can be obtained from them.

The alternative constructor sets the slotType , slotDescription and duration only, as these are the defining attributes of the object and should not be changed. The exclamation mark ( ! ) in the top right hand corner, indicating an excption being thrown, will be explained when its implementation is described.

The startTime may change, for example if the slot has to be rescheduled, and so it has a setter method, setSlotStartTime().

There are five inquiry methods corresponding to the five attributes indicated above, the getSlotEndTime() method will compute the endTime every time it is called.

The sixth inquiry method is called getSlotTypeAsString() which will return a string description of the slot type which is stored in the class as a manifest value, as will be described.

The toString() method will return a description of the slot as shown above.

The default constructor will produce a slot whose toString() will show it as follows. 

Unknown Unknown 
Start    0 min 0.0 seconds
Duration 0 min 0.0 seconds
End      0 min 0.0 seconds

The slotType attribute is a manifest integer, which will be described in detail when the RadioSlot implementation is described. The slotDescription is a String and the startTime and duration attributes are instances of the PlayTime class which is an extension of the BasicTime class, which will be described next.
