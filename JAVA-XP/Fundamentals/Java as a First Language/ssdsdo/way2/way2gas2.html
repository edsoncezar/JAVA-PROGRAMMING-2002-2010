When a class is member of a package this must be the first non-comment declaration in the source code file. Line 0010 states that the class declared in this file is a member of the gasbill package of classes. Line 0012 states that the MeterReading class extends the Object class, as shown in the class diagram. 

0014  private String address = "Unknown";
0015  private int    reading = 0;

The address attribute is declared as an instance of the String class with the initial value "Unknown". The String class is supplied by the Java environment and is capable of containing a sequence of characters. The reading attribute is declared as a variable of the primitive integer type with the initial value 0. Integer values are numbers that have no decimal parts (e.g. 18203, 0, -2367 etc.).

 

0017     public MeterReading( String theAddress, 
0018                          int    theReading) { 
0019        super();
0020        address = new String( theAddress);
0021        reading = theReading; 
0022     } // End MeterReading constructor.

The constructor takes two arguments, of the appropriate types, and stored their values in the two attributes; after the super, Object, default constructor has been called. (The MeterReading class does not have default constructor.) In order to store the value of the theArgument String in the address attribute a new String instance has to be constructed, using theAddress as an argument to the String constructor. The effect is to place the instance into a well defined initial state as defined by the values of the arguments. 

0025     public String getAddress() { 
0026        return address;
0027     } // End The getAddress.
0028  
0029     protected int getReading() { 
0030        return reading;
0031     } // End The getReading.

The two inquiry methods are declared in accord with the class diagram and are implemented by returning the value of the appropriate attribute. 

0034     public String toString() { 
0035        return this.getAddress + " reading " + 
0036               this.getReading();
0037     } // End toString.
0038  
0039  } // End MeterReading.

The toString() method returns a String which is obtained by combining the address and reading attributes, obtained by means of this instances inquiry methods, separated by the word reading. The string returned will be of the required form: form "10 Downing Street reading 18203".

The inquiry methods are used, rather than the alternative of accessing the attributes directly as shown below, due to an implementation rule of always using any available inquiry methods rather than direct access of attributes. (This allows the number and/or types of attributes to be changed, with appropriate changes to the inquiry methods, without disturbing methods such as this.).

0034     public String toString() { 
0035        return address + " reading " + 
0036               reading();
0037     } // End toString.

0039  } // End MeterReading.
