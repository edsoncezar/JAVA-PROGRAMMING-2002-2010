The BasicTime implementation

The implementation of the class as far as the start of the default constructor, is as follows. 

0001  // Filename playtime\BasicTime.java
0002  //
0003  // A class which records a time to an accuracy 
0004  // of one tenth of a second, supplying methods 
0005  // to obtain mins, secs and tenths in the format 
0006  // mm:ss:t.
0007  //
0008  // Produced for waypoint 2 99/00.
0009  //
0010  // Fintan Culwin, v0.1, sept 99.
0011  
0012  package playtime; 
0013  
0014  public class BasicTime extends Object { 
0015  
0016  private static final int TENTHS_PER_SECOND = 10;
0017  private static final int TENTHS_PER_MINUTE = 
0018                              TENTHS_PER_SECOND * 60;
0019  
0020  private int duration = 0;

Following the header comments, the package and class declarations lines 0016 to 0018 declare two constants which will be used in the arithmetic manipulation within the class. Line 0020 declares the duration attribute. The two constructors are implemented as follows. 

0022     public BasicTime() { 
0023        this( 0);
0024     } // End BasicTime
0025  
0026     public BasicTime( int tenths) { 
0027        super();
0028        duration = tenths;
0029     } // End BasicTime.

The implementation of these constructors is comparable to those of the Temperature class. The getMinutes() inquiry method is implemented as follows. 

0031     public int getMinutes() {
0032        return duration / TENTHS_PER_MINUTE;
0033     } // End getMinutes

The slash, / , symbol indicates division. As duration and TENTHS_PER_MINUTE are both integers, which will be an integer operation. For example if the value of duration is 1359, the operation would be 1359 divided by 600 which would return 2.

The getTenthsOver() inquiry method is implemented as follows.

0035     public int getTenthsOver() { 
0036        return duration % TENTHS_PER_SECOND; 
0037     } // End getTenthsOver

The percent, %, symbol indicates remainder after division. As duration and TENTHS_PER_SECOND are both integers, which will be an integer operation. For example if the value of duration is 1359, the operation would be 1359 divided by 10, which would return 135 remainder 9, and so the value returned would be 9.

The getSecondsOver() inquiry method is implemented as follows.

0039     public int getSecondsOver() { 
0040        return ( duration % TENTHS_PER_MINUTE) / 
0041                            TENTHS_PER_SECOND; 
0042     } // End getTenthsOver

The round brackets, ( ), indicate that that part of the operation should be done first so the expression ( duration % TENTHS_PER_MINUTE) would be evaluated first as (1359 % 600) which is 2 remainder 159, so the expression would evaluate to 159. This value is then fed into the / TENTHS_PER_SECOND part which is 159 / 10 which is 15.

The protected getDuration() inquiry method is implemented, without comment, as follows.

0044     protected int getDuration() { 
0045        return duration;
0046     } // End getDuration.

The final method, toString(), makes use of the public inquiry methods to produce the representation of the value of duration, as follows. 

0048     public String toString() {    
0049        return this.getMinutes()     + " min " + 
0050               this.getSecondsOver() + "." + 
0051               this.getTenthsOver() + " seconds";
0052     } // End toString
0053  
0054  } // End BasicTime.

As has been explained, if the value of duration is 1359: getMinutes() will return 2, getSecondsOver() will return 15 and getTenthsOver() will return 9. When these are catenated together as a string it would produce 2 min 15.9 seconds.
