As the two classes are in the radioslot package the two source code files have to be stored in a subdirectory, called radioslot, of the same directory which contains the playtime subdirectory.

Extend the RadioSlotDemonstation program so that it constructs and shows an instance of each possible type of slot ("Music", "Advert", "News", "Weather" or "Unknown").

The RadioSlotDemonstation program has demonstrated that an exeception will be thrown if the type argument to the alternative is less than UNKNOWN. The exception should also be thrown if the type argument is greater than WEATHER, adapt the demonstration program to show that this is the case.

This class diagram describes a PizzaOrder in a fast food pizza bar. The pizzaType attribute should be a manifest with possible values "margarita", "four cheeses" or "nettuno". The orderTime is a PlayTime instance indicating when it was ordered and the serveTime the time when it was served. The getTimeTaken() method will return the difference between the orderTime and the serveTime or 0 min 0.0 seconds if the pizza has yet to be served. The getPizzaTypeAsString() will return a string description of the pizza and the toString() method may produce a string in the following format.

Margarita pizza
Ordered at 45 min 49.8 seconds
Served at 0 min 0.0 seconds
Time Taken 0 min 0.0 seconds		 Four Chesses pizza
Ordered at 48 min 35.6 seconds
Served at 55 min 19.5 seconds
