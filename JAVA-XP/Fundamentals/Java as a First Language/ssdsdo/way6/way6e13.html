The TimerDemonstration class

The timer demonstration class provides an initial indication that the Timer seems to be operating. The first part of its implementation is as follows.

0014        System.out.println( "Timer demo ... starting the timer.");

0017                            aTimer.getTime() + 
0018                            " tenths of a second.");

This part of the demonstration constructs an instance of the Timer on line 0012 and subsequently starts it on line 0015. The number of tenths of a second already counted is then output on lines 0016 to 0018. The output from this part of the demonstration (when executed on a desktop pc) was as follows: 

Timer demo ... starting the timer.

Started ... it has already counted 0 tenths of a second.

The thread which is running the main() method of the TimerDemonstration program has been busy counting up to one million but, apparently at the same time, the thread running the Timer instance aTimer has also been busy as it has counted three tenths of a second.

The first part of this fragment again has the main() method counting up to one million but as the Timer thread is suspended (on line 0031) the Timer is still reported as having counted three tenths of a second. When the Timer is resumed, on line 0041, and main() counts again a further four three tenths of a second are counted by the Timer.
