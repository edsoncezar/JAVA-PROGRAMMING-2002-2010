Way6ADemo.java

The Way6A class contains three attributes and a rule which defines the valid state of the instances. The method which write the state of an instance to a DataOutoutStream and/ or that which reads the state of an instance from a DataInputStream is faulty.

The Way6ADemo program constructs a list of five Way6A instances, shows then to the user, writes them to a file, reads them back and shows them to the user again. The method called saveTheList() is incomplete (all other parts of the file are complete and correct and should not be changed!).

The Way6ADemo program and the Way6A class can be compiled and executed in thier current state.

Complete the Way6ADemo saveTheList() method first. When you have done this the program will read back and show five Way6A instances, but they will have the wrong state.

Correct the Way6A class so that it saves and loads instances correctly.
