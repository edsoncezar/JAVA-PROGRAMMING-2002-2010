Concurrency

All of the artefacts that have been introduced so far have had a single thread of control. That is Java has started execution at the start of the main() method and has followed through from that point doing only one thing at a time. It is also possible, and in most systems necessary, for more than one thread (of control) to be active at any instant.

For example when using a modern word processor: one thread may be responding to the keyboard and placing the characters on the screen, while a second thread is printing a document, a third thread is performing an auto-save, a fourth thread is checking the spelling, a sixth thread is checking the grammar, etc.

When two or more threads are executing at the same time they are said to be concurrent and the general term for this topic is concurrency. On a single processor architecture (such as a desktop pc) this is accomplished by the processor switching between the threads and giving the impression that more than one thing is happening at the same time. On multi-processor architectures different threads may be executing on different processors.

The major problem for concurrent programming is to ensure that the actions of the threads are synchronised, so that one thread does not interfere with the correct operation of another thread. This part of the waypoint will only provide an initial introduction to using more than one thread in Java, a more detailed introduction to this topic can be found in Chapter 8 of the recommended text book.
