The Timer run() and other inherited actions

As the Timer class extends the Thread class it inherits a number of methods including:

public synchronized native void start()
public final void stop();

These two actions start and stop the execution of a thread, once a thread is started its run() method is called in a concurrent manner and continues until it terminates or until the thread's stop() method is called. (The modifier native indicates that this action is implemented by the underlying operating system, not Java. The final modifier indicates that this action cannot be overridden. Neither of these details need be considered important at this stage).

Starting and stopping a thread are expensive operations, two other methods are supplied if all that is required is that the thread does not execute for a period of time.

public final void suspend();
public final void resume();

A thread which has been suspended will not continue execution until it is resumed. It is also possible to suspend the thread for a certain amount of time and have it resumed automatically by using its sleep() method.

public final void sleep( long milliseconds) 
                         throws InterruptedException;

Calling sleep() will suspend the thread for (approximately) the number of milliseconds specified. It is possible that the thread will be woken up before this time (for example if the application is about to terminate) in which case an InterruptedException is thrown.

In general: in order to make simple use of a thread all that needs to be done is to override the run() method in a class which extends Thread. The implementation of the Timer run() method is as follows.

0018     public void run() { 
0019        while ( true) { 
0020           try { 
0021              this.sleep( 100);
0022           } catch (InterruptedException exception) { 
0023              // Do nothing. 
0024           } // End try/ catch.
0025           theTime++;        
0026        } // end while.
0027     } // End run.

The run() method of a Thread is called from the start() method and the thread will continue execution until the run() method finishes (or until the thread is otherwise terminated). To keep a thread running indefinitely a non-terminating while loop can be used, as in lines 0019 to 0026. Within the loop the thread is put to sleep for 100 milliseconds (one tenth of a second), on line 0021, and when it wakes up, on line 0025, it counts one tenth of a second by incrementing theTime attribute.
