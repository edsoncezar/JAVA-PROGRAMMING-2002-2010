Event Driven Programming

In this part of the Waypoint a Graphical User Interface (GUI) will be provided for the Timer class previously described. To prepare for this the Timer class will be redeveloped to produce the TimerEventSource class. This class will generate and dispatch an event every tenth of a second.

An event is a Java Object of the Event class (or one of the classes extended from it). It is a means of passing a message from one class to another indicating that something significant has occurred. In this example the TimerEventSource class will generate an ActionEvent instance every tenth of a second and an attribute of the instance will indicate how many tenths of a second have elapsed since the timer was started, or last reset.

The ActionEvent instance generated by the TimerEventSource instance will, in the first artefact, be output onto the terminal screen. This will give an initial indication that the TimerEventSource class is operating correctly and will also introduce the techniques by which Events can be received and their attributes inspected.

The second artefact will introduce the techniques by which a window can be placed upon the desktop. It will do no more than continually show the value received from the timer and will not attempt to interact with the user.

The third artefact will provide the user with three Buttons that allow the timer to be used as a simple (and not very accurate) stopwatch. This will allow the techniques by which graphical user interfaces can be produced to be introduced. The Buttons on the interface will themselves produce ActionEvents when the user interacts with them. This can only be an initial introduction to these techniques, further details will be given in subsequent units.
