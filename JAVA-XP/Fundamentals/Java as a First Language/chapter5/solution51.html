Exercise 5.1

The RoomMonitor class as implemented at the start of this chapter is a WarningMonitor class, but did not have its warning behaviour demonstrated. Adapt the demonstration client to allow this aspect of its behaviour to be demonstrated.

The default constructor of the RoomMonitor class does not require any arguments when instantiated but will pass two arguments to its parent class. The two arguments are (0, MAX_VALUE), which will set the range of the counter to be all positive integer values. This means that in this class only the unCount() action can cause an exception to be thrown ( even though it is still recognized that the count() action can also throw an exception, because there is no upper limit to the range of the counter, it is not possible in this instance).

To throw an exception the number of people in the room must count below 0. So far, the RoomMonitorDemonstration has counted 3 people in the room so therefore the exitRoom() action must be called 4 times.
