Pedagogic criteria


This part of the activity asked the attendees to nominate, and subsequently evaluate the importance of, various criteria to evaluate a language's pedagogic suitability against. The table shows in the first column the name of the criteria, linked to a more complete explanation. The following columns are the number of responses received and the average score, in the range 0 to 5, where 0 indicates irrelevance and 5 that it is of overwhelming importance. The final column is the standard deviation indicating the degree of consensus with smaller values indicating closer agreement.

One criteria from the workshop has been split into two criteria on this table. The criteria hybrid indicates a hybrid language, that is one which actively supports both procedural/ imperative and object oriented styles of development. The criteria pure indicates a language which only actively supports an object oriented development style. During the workshop these were presented as a single construct with a negative score indicating purity. This seems to have confused many of the attendees (sorry!) and thus fewer responses were made, although a number of attendees indicated that they thought the criteria to be irrelevant.

stability & standard refers both to the existence of an ANSI and/ or ISO standard for the language and the degree to which the language is not subject to change.

strong typing is the extent to, at compile time and at run time, only actions appropriate to an object are applied to it.

simplicity is a combination of the syntax's simplicity, regularity and brevity.

pedagogic support is the availability and quality of resources to support the language's use within the curriculum.

cross curriculum refers to the language's suitability for use in all areas of the curriculum as opposed to its suitability for use as first/ base language.

cost refers to the costs involved in using the language, both for students and for the institution.

learning curve refers to the initial learning curve for the language specifically, but not exclusively, the amount of syntactic overhead involved in producing an initial artifact.

safety refers to the avoidance of gotchas. That is (syntactic) constructs in the language which will impede a student developing a clear conceptual model of its operation. (The archetype for this must be the confusion between = and == in C/ C++ control expressions.)

marketability refers to the initial commercial value of student's experience with the language.

hybrid and pure refer, as explained above, to the extent to which the language does (hybrid) or does not (pure) support styles of development other than OO.

fun (I am not sure what this means, apparently students in the USA are expected to enjoy the experience of learning software development!)

conceptual expressability refers to the simplicity and regularity of the cognitive connnections between a language's syntax and semantics.
