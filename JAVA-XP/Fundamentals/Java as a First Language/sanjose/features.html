details

polymorphism was defined as the extent to which the language supports polymorphic methods and overloading of methods and operators.

generics was defined as the extent to which the language provides facilities for generic instantiations of templates.

encapsulation was defined as the extent to which the language allows information to be hidden within an object's implementation.

inheritance was defined as the extent to which the language supports development by extension, allowing an object's implementation to make use of an existing object's implementation.

reuse was admitted not to be a defining feature of OOP but was included as an essential feature of any language. It was defined as the ease with which source or object code could be successfully reused.

simple was, contentiously, defined as a combination of the syntax's simplicity, regularity and brevity in combination with the semantic meaning of the language's terms.

type safety was defined as the extent to which the language, at compile time and at run time, ensures that only actions appropriate to the object are applied to it.

memory management was defined as the extent to which the language removes from the developer responsibility for dynamic allocation, and de-allocation, of resources as objects are created and destroyed.

interface & implementation was defined as the extent to which the language allows what an object does to be separately stated from details of how an object does what it does.

Commentary

Not surprisingly C++ was the most well known and least favored language which raises the question if its low rating is due to its inherent features of because the attendees were more familiar with its shortcomings. Correspondingly Eiffel was the least well known most favored language.

Of the big three languages Java was the most favored but, when processing the raw data from the forms some evidence of a halo effect for Java and a devil effect for C++ was noted. Thus although the template mechanism in C++ provides a more elegant and powerful generic mechanism than the generic facilities in Java, it was less favorably assessed.
