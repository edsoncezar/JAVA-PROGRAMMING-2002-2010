Exercises for Chapter 4

  • 4.1 Complete the design and implementation of the StoppingCounter and WarningCounter demonstration clients.

  • 4.2 Produce a diagram similar to that given in Figure 3.5 to describe the actions of one of the Counter classes in this chapter.

  • 4.3 Re-design, re-implement, and re-demonstrate the class hierarchy with the WarningCounter as a child class of the StoppingCounter.

  • 4.4 Extend the hierarchy from Exercise 4.3 adding a child class of the WarningCounter class, called InfoWarningCounter class which makes the isAtMaximum and isAtMinimum actions public. Reimplement the demonstration client so that it uses these actions to avoid throwing an exception.

  • 4.5 Implement and demonstrate the specialised counters from Exercises 3.3 and 3.4.

  • 4.6 Extend the Employees hierarchy from Exercise 3.4 to include a VoluntryEmployee class as a child class of the WagedEmployee class. If any attempt is made to access the enquiry action which returns the value of the hourly wage attribute a suitable exception should be thrown. Design, build and demonstrate this class.
