3.8 Demonstrating the LimitedCounter class

As with the demonstration of the BasicCounter class, the LimitedCounter class will have to be temporarily implemented as a public non-abstract class with public methods. Only a partial demonstration of the LimitedCounter class, without a design, will be presented here. A full demonstration and the production of designs for it will be left to an end of chapter exercise.

The first part of the demonstration client, as far as the output of the title and the creation of an LimitedCounter instance, might be as follows.

The call of the principal constructor on line 0022 has to specify the lower and upper limits of the range and in order to simplify the demonstration these delineate a very small range between 10 and 12. The call on line 0029 of the BasicCounter numberCountedIs() action is permissible as the LimitedCounter instance denoted by aCounter is an extended instance of the BasicCounter class and thus inherits all of its public actions as public actions.

The call of the LimitedCounter isAtMinimum() action on line 0033 returns a value of type boolean. This value is immediately output by the System.out.println() action which will output "true" if the value is true and "false" if it is false. As shown on the output the value at this stage of the demonstration is true. This is the expected output as the previous call of numberCountedIs() indicates that 10 occurrences have been counted and the call of minimumIs() indicates that 10 is the minimum value of the range.
