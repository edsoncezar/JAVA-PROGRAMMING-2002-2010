These actions are called when the user selects a date in the MonthPanel or one of the options on the Choice menus respectively. The actionPerformed() action will not require the monthPanel to be reconfigured and indirects immediately to the dateSelected() action to cause the DatePanel to generate an ActionEvent.

A call of the itemStateChanged() action indicates that the user has changed one of the Choice menus and so a different year or month has been selected. This requires the reConfigure() action of the monthPanel to be called before the dateSelected() action is called.
