The DatePanelResources_fr class

Using the systematic naming convention, the French language resources have to be stored in a class called DatePanelResources_fr implemented as follows.

0001  // Filename DatePanelResources_fr.java.
0002  // Contains the French language resources 
0003  // for the DatePanel class.
0004  //
0005  // Written for Java Interface book chapter 9.
0006  // Fintan Culwin, v 0.2, August 1997.
0007  
0008  import java.util.ListResourceBundle;
0009  
0010  public class DatePanelResources_fr extends ListResourceBundle { 
0011  
0012  static final String dayNames[]   = { "Lun", "Mar", "Mer", "Jeu",
0013                                       "Ven", "Sam", "Dim"};
0014  
0015  static final String monthNames[] = { "Jan", "Fév", "Mar", "Avr", 
0016                                       "Mai", "Jun", "Jul", "Aoû",
0017                                       "Sep", "Oct", "Nov", "Déc"};
0018                                       
0019     static final Object[][] contents = {    
0020        { "dayNames",   dayNames },      
0021        { "monthNames", monthNames },            
0022     }; // End contents.
0023
0024     public Object[][] getContents() { 
0025        return contents;   
0026     } // End getContents.
0027  
0028  } // End class DatePanelResources_fr.

If this file and the default DatePanelResources file are both compiled and the resulting class files stored in the same directory then the DatePanelResources_fr ResourceBundle will be opened if the system properties indicate a French linguistic environment and the default DatePanelResources ResourceBundle in other environments.
