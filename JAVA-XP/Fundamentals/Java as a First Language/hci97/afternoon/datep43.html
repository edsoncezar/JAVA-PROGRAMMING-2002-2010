The DatePanelResources class

The implementation of the DatePanelResources class is as follows. 

0001  // Filename DatePanelResources_fr.java.
0002  // Contains the French language resources 
0003  // for the DatePanel class.
0004  //
0005  // Written for Java Interface book chapter 9.
0006  // Fintan Culwin, v 0.2, August 1997.
0007  
0008  import java.util.ListResourceBundle;
0009  
0010  public class DatePanelResources extends ListResourceBundle { 
0011  
0012  static final String dayNames[]   = { "Mon", "Tue", "Wed", "Thu",
0013                                       "Fri", "Sat", "Sun"};
0014  
0015  static final String monthNames[] = { "Jan", "Feb", "Mar", "Apr", 
0016                                       "May", "Jun", "Jul", "Aug",
0017                                       "Sep", "Oct", "Nov", "Dec"};
0018                                       
0019     static final Object[][] contents = {    
0020        { "dayNames",   dayNames },      
0021        { "monthNames", monthNames },            
0022     }; // End contents.
0023
0024     public Object[][] getContents() { 
0025        return contents;   
0026     } // End getContents.
0027  
0028  } // End class DatePanelResources_fr.

The DatePanelResources class extends the ListResourceBundle class which supplies the actions required for the I18n mechanism to find it. It need only declare a two dimensional array of Objects, in this example called contents. The first dimension of the array contains the keys which will be used to retrieve the resources. The second dimension contains the resource objects themselves.

In this example the two keys are Strings with the values "dayNames" and "monthNames". The corresponding resources are one dimensional arrays of Strings with the values shown.

The class need only override the getContents() action to return the identity of the a two dimensional array of Objects.
