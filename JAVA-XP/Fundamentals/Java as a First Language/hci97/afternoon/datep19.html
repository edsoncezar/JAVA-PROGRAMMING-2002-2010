The DatePanel class - class diagram 2

The DatePanel has to receive events generated by its century, decade, year and month Choice components and so it has to implement the ItemListener interface, supplying the itemStateChanged() action. It also has to listen to ActionEvents generated by its MonthPanel and so supplies an actionPerformed() action.

As well as being the recipient of events from its contained components it is also the source of ActionEvents whenever its date is changed and, in order to be compliant with other components supplies addActionListener(), removeActionListener(), setActionCommand() and getActionCommand() actions. This allows it to register multiple listeners and all will have an event dispatched to them when appropriate.

The default constructor provides a panel with the current system date, the alternative with the date as supplied by its arguments. Once constructed the date can be changed by the setDate() action and the three remaining actions allow each constituent of the date to be enquired.
