The DatePanel is a source of ActionEvents (or should it be ItemSelectedEvents?) which requires it to support an actionCommand interface consisting of setActionCommand() and getActionCommand() actions. These actions manipulate the value of an encapsulated String called actionCommand as shown above.

This is identical to other ActionEvents sources, such as the Button class, and the current value of the actionCommand will be passed as a part of the ActionEvents generated by a DatePanel whenever its date is changed. This design pattern allows a developer to use their knowledge of ActionEvent sources when using the DatePanel and not have to learn a new pattern of actions.

In addition to the actionCommand actions an ActionEvent source is also expected to supply addActionListener() and removeActionListener() actions to allow multiple listeners to be registered and called.
