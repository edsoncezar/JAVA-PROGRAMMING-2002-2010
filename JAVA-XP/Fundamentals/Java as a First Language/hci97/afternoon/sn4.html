The MessageCanvas class - class diagram

The MessageCanvas class can be thought of as a (simplified) multi-line version of the Label class.

Its constructor requires the message which it is to display as an argument and, once constructed, is immutable. The addNotify() action will be called when its component peer has been constructed and information relating to the actual environment which it is operating within can be obtained. The paint() action is supplied in order for the instance to be able to present itself on the screen and the getMinimumSize() and getPreferredSize() are required for it to take part in geometry negotiations.

Geometry negotiations are performed when a Panel lays out its children. Each is asked in turn for its preferred size and if these can be accommodated within the Panel's layout management policy and its available size, will be granted. Otherwise their minimum sizes will be asked for and possibly granted. A component should not assume anything about the amount of space which it has been allocated.
