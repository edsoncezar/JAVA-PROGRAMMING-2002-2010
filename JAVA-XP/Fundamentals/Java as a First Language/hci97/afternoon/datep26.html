The AWT recognises that Component developers will have a requirement to multicast events to a number of listeners and provides a mechanism to facilitate this. The AWTEventMuticaster class provides class wide actions which will add, remove and dispatch events to a set of listeners on behalf of the developer.

In this example the itsListener attribute is an instance of the ActionListener class, declared in the header on line 0026, and the add() and remove() actions will allow listeners to be registered and de-registered with it. It is used in the MonthPanel dateSelected() action, called whenever the DatePanel should generate an ActionEvent, as follows. 

0157      private void dateSelected() { 
0158         if ( itsListener != null) {   
0159            itsListener.actionPerformed( new ActionEvent( this, 
0160                                             ActionEvent.ACTION_PERFORMED,
0161                                             this.getActionCommand())); 
0162        } // End if.                                              
0163     } // End dateSelected.

If there are any listeners registered then the value of itsListener will not be null and by calling its actionPerformed() action a copy of the ActionEvent, whose ID is set to ACTION_PERFOMED and whose actionCommand is set to that provided by the call of getActionCommand(), will be sent as an argument to the actionPerformed() action of all registered listeners. There is no guarantee on the sequence in which these action will be called.
