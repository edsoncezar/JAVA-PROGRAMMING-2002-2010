L10n and I18n - overview

The java.util.ListResourceBundle supplies the basic mechanism for the localisation and internationalisation of a Component. The mechanism relies upon the user.language and user.region system properties to automatically decide which particular bundle to obtain the resources from.

The developer extends ListResourceBundle to define a class which contains the default (USA English?) resources.

The developer then supplies alternatives to the default bundle, using a systematic naming convention, containing the same resources with their the localised and internationalised alternatives.

At run time the Java environment will retrieve the user.language and user.region system properties and construct a name for the resource bundle based upon the name of the default bundle, using the same systematic rule.

If a resource bundle with the systematised name can be located it will be opened and its contents used. Otherwise the default bundle will be opened and used.

For example a component called Foo may have its default resources in a bundle called FooResources. For a French speaking user in Canada the user.language will be fr and the user.region will be CA. The mechanism will first look for FooResources_fr_CA then FooResources_fr and finally FooResources.
