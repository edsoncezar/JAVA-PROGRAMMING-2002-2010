This is a non-public (hence private) class contained within the same source code file as the canvasExample class. This is the mechanism for encapsulating one class within another.

The constructor commences by calling its parent (Canvas) constructor referring to it as super. It then resizes itself and enables MOUSE_MOTION_EVENTs and MOUSE_EVENTs upon itself. It is this action which causes suitable events to be dispatched to itself.

The addNotify() action is called after the instance has been actually created in the peer environment. The peer environment is the actual environment ( Motif, Windows, Macintosh, etc.) which the applet is executing within. It is only at this stage that specific information, such as its graphics context, can be obtained. A graphics context is needed to draw upon the component's window.

This addNotify() action first calls its parent (super) addNotify() action and then creates an encapsulated Graphics (context) instance called context.
