Interface Design by Bill Venners
Chapter 3
interfaces and Abstract Classes
Interface Design | Contents | Previous | Next

The Guidelines:

  • Guideline: Appreciate the significance of the interface.
  • Guideline: Use interfaces to say "What objects can do for you" or "What you can do with objects".
  • Guideline: Use interface extension to decouple services or restrict the semantic contract.
  • Guideline: Consider creating convenience implementations of your interfaces.
  • Guideline: Use abstract classes to provide default behavior or define a family.
