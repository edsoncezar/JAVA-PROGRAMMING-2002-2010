Defining HTML Documents to Run Applets

The subject of defining HTML documents can fill an entire book. The goal in this section is to present only sufficient elements of HTML to show how applet specifications can be embedded in an HTML document.

On the CD
Listing 8.2 (file HelloAppletWorld.html on the CD-ROM accompanying this book) shows an example of an HTML document for running the HelloWorld program defined in the previous section. The line numbers in this listing are for reference purposes only, and are not part of the HTML document.

HTML documents consist of tags enclosed in angular brackets: 

<tag>

A tag is used to specify the type of data that is encoded in the document. Most tags have an end-of-tag specifier that marks the end-of-tag definition. These end-of-tag specifiers have the following format: 

</tag>

Spaces and lines outside a set of tag specifiers are usually not significant.

Lines 1 and 15 of listing 8.2 define the tags <HTML> and </HTML>, respectively. These tags specify that everything between these tag specifiers is part of an HTML document.

Lines 2 and 4 of listing 8.2 define tags <HEAD> and </HEAD>, respectively. These tags specify the head of the HTML document. Within this head specification is the <TITLE> and </TITLE> tags on line 3. Inside the <TITLE> specification is the string "A simple HelloWorld Applet". This title will be displayed as the title of the HTML document in the Web browser.

Lines 6 and 14 of listing 8.2 define tags <BODY> and </BODY>, respectively. These tags specify the body of the HTML document that will be rendered as a graphics document in the Web browser.

On line 7 is the tag specifier <P> that defines a new paragraph. The end </P> tag specifying the end of the paragraph has been left out because it is optional.

The tag <BR> on line 8 causes a line break, and displays the contents that follow on a new line. Lines 10_13 define the applet. Note that these contain the <APPLET attributes> and the end of applet specification tag </APPLET>.

The attributes in the applet tag can contain a number of attribute specifications. In this example, only three attributes, CODE, WIDTH, and HEIGHT are defined. A number of other attributes can also be specified. These are discussed in later examples in this chapter.

Page 289

The CODE="URL" attribute specifies the URL location of the applet code. This URL location specifies the file that contains the applet class. Note that this file contains the byte code for the applet and must be specified by using the .class extension. In this case, the class file is in the same directory as the HTML file. If the class file is in a different directory than the HTML file, you can use the CODEBASE="directory" attribute to specify the different directory.

The WIDTH and HEIGHT specify in pixels the width and height of the bounding rectangle that contains the applet. It is used to specify how big a box to draw for the applet. You should specify appropriate sizes for the applet, taking into account how big a graphics object is painted by the applet. The resize() method can be called from within the applet code to resize these dimensions.

The text between the <APPLET attributes> and </APPLET> tags is optional. If specified, this text is displayed by non-Java-enabled browsers. In the example in listing 8.2, the following statement will appear in non-Java-enabled browsers, where the applet would normally be displayed:

You will not know what you are missing unless you upgrade your browser to a Java-enabled browser! You should display a similar message (probably a less obnoxious one!) to indicate the fact that the applet cannot be run in the user's browser.

Running Applets

You can use the Applet Viewer application included with the JDK or any Java-enabled browser to run the applet.

On the CD
You can run the applet with Applet Viewer by using the following steps:

  1. After you have copied the contents of the HelloAppletWorld directory from the CD-ROM (at the back of this book) to your local disk, you should build the .class file by running the Java compiler.
  2. From the directory in which you copied the code, type javac HelloAppletWorld.java.
  3. After the program compiles with no errors, type appletviewer HelloAppletWorld.html. The Applet Viewer should launch, and you will see the applet display as shown in figure 8.7.

You can also start the applet in a Web browser such as Internet Explorer. You can
