Defining Class Variables

As you learned in the previous section, class instance variables for separate objects have a separate existence. A class variable or a class wide variable, on the other hand, is common for all class instances. These class variables are preceded with the reserved word static: 

static type variable = [initialize_value];

Page 183

Suppose that you want to keep track of the number of times a room has been occupied, and also track the number of times repairs were done to the room. You cannot 1effectively use instance variables to keep track of these values because instance variables are defined for a particular class instance only, and you need global counters for all class instances.

Consider an extension to the class Room: 

class Room
{
      // Class instance variables
      int maxRooms;
      int entranceType;
      float area;
      boolean bathroom;


      // Class variables
      static int nOccupancies;
      static int nRepairs = 0;

      // Other variables.

      // Other methods

}

The variables nOccupancies and nRepairs are class variables because they are preceded by the reserved word static. Both variables are of type int, but one of them (nRepairs) has an initial value of 0. The variable nOccupancies is used to keep track of the number of times the room has been occupied, and the variable nRepairs is used to keep track of the number of repairs to the room.

Consider the two instances rm1, rm2 of the previous class. How would you reference the variables nOccupancies and nRepairs? You cannot reference them by using the following syntax: 

rm1.nOccupancies
rm2.nRepairs

This attempt to reference the class variables is incorrect, because the class variables are common to all instances of the class. Figure 6.3 shows the difference in storage space for class instance and class variables.

To correctly reference class variables, you must use the following syntax:

classname.variablename

Therefore to reference the class variable nOccupancies and nRepairs, you must use the following syntax: 

Room.nOccupancies
Room.nRepairs

Because the data area storage for class variables is the same for all class instances, changing a class variable will change it for all class instances.

Class Constants

Suppose that you want to define an integer array of size 312. You can use a constant literal to specify the size of the array: 

int[] xa = new int[312];

But this is not always good programming practice because the constant values (312 in the preceding example) appear to be like "magic numbers," and it is harder to figure out why a particular constant value is used. For this reason, a variable name that is to be treated as a constant is generally preferred. When defined within a class, these constants are called class constants. If, for example, the array size was defined by the constant MAX_SIZE with value 312, you could write the preceding array definition as the following: 

int[] xa = new int[MAX_SIZE];

Page 185

The preceding statement is not only more readable, but does not raise the question, as did the first version of this statement, as to why a value of 312 was used. You can more readily tell that the value 312 is the maximum size of the array.

You could define MAX_SIZE as follows: 

int MAX_SIZE = 312;

In this definition, however, MAX_SIZE is a variable whose value can be accessed and changed within the program—it is not really a constant. Java enables you to define symbolic constants or named constants called class constants, which can be referred to by an identifier by using the following syntax: 

final type variable = initialize_value;

The class constant definition is similar to a class variable definition with the exception of the reserved word final preceding the definition. The reserved word final indicates that the value assigned by initialize_value is the final value (constant value), and it cannot be changed. Any attempt to change the value of the class constant will generate a syntax error when the program is compiled. Thus, MAX_SIZE could be defined as follows: 

final int MAX_SIZE = 312;

Here is an example of the use of class constants: 

class ConstClass
{
      final int MAX_STUDENTS = 100;

      public void Ex1()
      {
         int[] grades = new int[MAX_STUDENTS];

         for (int x = 0; x < MAX_STUDENTS; x++)
      {
            grades[x] = 0;

         } // end for

         } // end Ex1

} // end ConstClass

This is not a severe limitation, as it may first seem, because methods always occur inside classes, and a class constant defined in a class is accessible by all methods in the class (see fig. 6.4).

Implementing Class Methods

Most of the logic for classes is contained in the class methods. The class variables are also important to define because they describe the data structures that are used in the class methods, but the behavior of the class is contained in the class methods.

When you implement classes, you will spend most of the time defining the methods and implementing their logic. The following sections focus on different aspects of defining class methods.
