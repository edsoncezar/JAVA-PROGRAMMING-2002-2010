Method overriding is different from method overloading. In method overloading, you are providing methods with the same name but more than one signature, such as methods with different arguments. All the overloaded methods exist simultaneously. When you override a method, you are providing a method with the same signature as a method in the superclass. The overriding method provides an alternative implementation to the overridden method. The overriding method may invoke the overridden method in the superclass by using the reserved word super to refer to an instance of the superclass.

Using Methods in the Root Object

Lines 49_55 in the main() method in listing 7.3 invoke the toString() method to get a string representation of the objects ptxy1 and ptxyz1. This string representation is then printed out. Lines 51 and 55 in listing 7.3 print these objects using the println() method. The println() method uses the toString() method to print the object.

Figures 7.14 and 7.15 show the output of the program in listing 7.3. In figure 7.15 you see displayed the results of the call to toString(). As you examine listing 7.3 you will notice that there is no toString() method defined in classes S and E. Yet the program compiles and executes to produce the display in figures in 7.14 and 7.15. So where is the toString() method defined? You can find the answer to this question by examing the inheritance hierarchy diagram of figure 7.16.
