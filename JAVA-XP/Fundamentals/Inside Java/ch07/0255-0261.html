Controlling Access to Classes

In the examples of this chapter, the Java program code was contained in a single file. In general, when writing larger programs, you may want to split the Java code in separate files. The Java language has rules about which methods are accessible to other classes in the same file or a different file. These rules are discussed next.

Controlling Class Access Using Public, Protected,
Private, or Friendly

The reserved words public, protected, and private can be used to control the level of visibility of a class method or variable.

When you precede a method definition or variable with the reserved word public, the method or variable can be accessed from any class in any package. This includes classes defined in the same file as the class with the public method/variable or a class defined in another file. Classes defined in other files are part of a named package. Recall that a package is a collection of classes grouped together based on their logic and function. In the program examples of listing 7.4, if you wanted other packages to use the class S and E, you should declare their constructors as public:

If you want to make a method/variable in a class available to any subclass inherited from it, you should declare the method/variable as protected. Protected methods/variables are accessible to any subclass of the declaring class or a class in the same package as a declaring class. The instance variables in class S, for example, sx and sy (listing 7.4) could be declared as protected. Then these instance variables would be accessible in class E, even if the class E was defined in another package:

If you want to make a method/variable in a class inaccessible to any class, you should declare the method/variable as private. Private methods/variables are inaccessible to any other class, including the subclass of the declaring class or a class in the same package as a declaring class. The instance variable in class E, for example, ez (listing 7.4) could be declared as private, as there is no need for any outside class to access this variable:
