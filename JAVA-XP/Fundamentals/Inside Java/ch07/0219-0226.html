Object-Oriented
Programming with Java

In the previous chapter you learned how to define and structure
classes. In this chapter you will learn to use an important tech-
nique of programming that is generally lumped into a category called object-oriented programming.

Java supports object-oriented programming by using an inheritance mechanism that enables a class to extend another class. In this chapter you will learn about superclasses and inheritance, and how these mechanisms enable you to reuse logic in existing classes.

Inheritance in Java

This section will explain the basic properties of inheritance in relationship to the Java language.

Understanding Inheritance Terminology

When a class E extends a class S, a relationship of the type shown in figure 7.1 is set up between them. The class S is called the superclass, and the class E is called the subclass or extended class. Other terms that also have been used to describe this relationship refer to S as the parent class and refer to E as the derived class, or child class. Class E is also said to inherit from class S.

In the inheritance relationship shown in figure 7.1, class E, as well as being able to access its own instance variables and methods, also can use the methods and instance variables that class S has made available for just such a purpose.


Figure 7.1 Inheritance relationship.

In Java syntax, the relationship of figure 7.1 is described as the following: 

class E extends S
{
      // Definition for class E

} // end E

Notice that the use of the reserved word extends the definition of the superclass S.

Consider the following skeleton structure for the two classes:

Building a Derivation Chain

You can also define other classes that can inherit from class E, even though class E inherits from another class S, as seen in the previous section. This type of inheritance relationship is shown in figure 7.3. The diagram of figure 7.3 is also called a class hierarchy diagram. Class hierarchies describe the relationship between Java classes. Java's predefined classes use class hierarchy diagrams.


Figure 7.3 A three-level inheritance chain.

Consider another class F inherited from the class E, as shown in the following example: 

class F extends E
{
      // Declarations for instance variable fa, fb, fc.

      void f_method1()
      {
         …
      } // end f_method1

      void f_method2()
      {

All the methods in class F can, of course, use their instance variables fa, fb, fc, and methods f_method1(), f_method2, f_method3(). In addition, these classes can also use the instance variables in class E, ea, eb, and ec; and also instance variables in E's superclass sa, sb, and sc. Additionally, the code in F can also use the methods in class E and the superclass S; that is, e_method1(), e_method2(), e_method3(); and s_method1(), s_method1(), and s_method3().

The relationship between classes S, E, and F is also shown in figure 7.4. The inherited class F is shown as encapsulating the class E, which is shown encapsulating S. This indicates that the methods and instance variables in E and S are available in class F.
