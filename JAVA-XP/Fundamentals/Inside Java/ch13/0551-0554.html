The Java AWT has five layout managers, each of which can implement the LayoutManager interface. The following list organizes these layout managers from simplest to most complex:

  • FlowLayout
  • BorderLayout
  • CardLayout
  • GridLayout
  • GridBagLayout

Other layout managers are available from Java component vendors and Java Web sites. You also can create your own layout managers.

One or more layout managers can be used to lay out a container. This is because containers can hold other containers, and each of these containers can have a different layout manager. Table 13.17 describes some characteristics of each AWT layout manager.

BorderLayout Easy to use. Has five areas in which to place components: North, South, East, West, and Center. Good for laying out dialog boxes that have buttons across the bottom. In this case, the Center area usually contains a Panel with another layout manager. Also good for "quick and dirty" applets.
CardLayout A little harder than the previous two. One "card" (which can contain a component or container) is shown at a time. Can be used for implementing notebook-style (tabbed pages) functionality.

Now that we introduced the Java AWT layout managers, let's examine each one in more detail.

FlowLayout

Some examples that you have already seen in this chapter use FlowLayout to lay out components. Take another look at listing 13.1 (the HelloGuiWorld applet) and the associated display in figure 13.2. This applet creates a button and a label, and adds them to the applet's display. Recall that the Applet class is derived from the Panel class; the default layout manager for a Panel is FlowLayout. Therefore, because no layout manager is specified for the HelloGuiWorld applet, the default is FlowLayout.

Recapping listing 13.1, the button is created and added to the applet in lines 25
and 26: 


25           pushButton = new Button("Push");
26           add(pushButton);

Lines 36 and 37 create and add the label to the applet:

Page 553

 
36           greetingLabel = new Label(helloString);
37           add(greetingLabel);

As you can see, you simply call the add() method of the container to which you are adding the component. Because the FlowLayout layout manager has been (implicitly) set, it takes over the job of laying out the components. To explicitly set the layout manager of a container to FlowLayout, use the setLayout() method of the container with one of the following forms of the FlowLayout constructor: 

setLayout(new FlowLayout(int align, int hgap, int vgap));
setLayout(new FlowLayout(int align));
setLayout(new FlowLayout());

The first form of the constructor takes three parameters:

  • Alignment (can be LEFT, RIGHT, or CENTER). This controls where components will be added. If the components added exceed the width of the container, they will be added below the first row, using the specified alignment.
  • Horizontal gap between components in pixels.
  • Vertical gap between components in pixels.

The second form takes only the alignment parameter and defaults to horizontal and vertical gaps of five pixels.

The third form defaults to CENTER alignment and gaps of five pixels.

BorderLayout

For a good example of BorderLayout, look at listing 13.2 (the ScrollbarSlider applet) and the associated display in figure 13.5. BorderLayout is used in line 36 to stretch a Scrollbar (instance variable named slider) across the bottom of the applet: 

36           add("South", slider);

In lines 44_46, a Panel is added to the center area of the applet so that a TextField (instance variable named value) can be added to the Panel and displayed: 

44           Panel p = new Panel();
45           add("Center", p);
46           p.add(value);

If the text field had been added directly to the container in line 45, it would have stretched to fit the entire center area. The diagram in figure 13.10 shows the areas to which components can be added.


Figure 13.10 Diagram of BorderLayout areas.

To add a component to a container that is governed by a BorderLayout, use the following form of the container's add() method: 

add(String name, Component comp)

The String in this case is either "North," "South," "East," "West," or "Center." The Component is the component that you want to add to the container. The component will be stretched horizontally if added to the North or South, or vertically if added to the East or West. The component will be stretched both horizontally and vertically to fill the remaining space if added to the Center area.

To set the layout of a container to BorderLayout, use the setLayout() method of the container with one of the following forms of the BorderLayout constructor: 

setLayout(new BorderLayout(int hgap, int vgap));
setLayout(new BorderLayout());

The first form of the constructor takes two parameters: the horizontal gap and the vertical gap (in pixels) that will be placed between components.

The second form uses default values of zero for these gaps.
