So far, you've learned to use several Java GUI components. One more component will be explored in this section—the Canvas component.

The Canvas Component

The Canvas component is the simplest of all the components. This component is essentially a blank rectangular area in which the application can draw lines and shapes or display images. All the Graphics methods that you learned in Chapter 9, "Animating Applets with Java Graphics, Images, Colors, and Sound," can be used with the Canvas class. Because the Canvas class is derived from the Component class, it contains all the inherited capabilities, including appearance management and event-trapping.

To construct a new Canvas, use a no-argument constructor, as shown here: 

Canvas c = new Canvas();

For more information on the Canvas class, refer to the following Web page: 


http://java.sun.com/products/JDK/1.1/docs/api/java.awt.Canvas.html

You also can subclass the Canvas class to create your own custom components. You will learn how to create custom components in Chapter 15, "Java Beans: Java's Component Architecture."

Using AWT Containers

An AWT container is a specialized component that contains other components. Figure 13.1 shows that the Container class is derived from the Component class. Several types of containers are derived from the Container class—Panel, Applet, ScrollPane, Window, Frame, Dialog, and FileDialog.

Each container has a specialized purpose. A Dialog, for example, appears on the screen to display and gather input from the user. A ScrollPane is used to manage a scrolling area, and a Panel is used solely to organize other components. All share a common function, however—they contain and organize other components. These components are usually organized with the help of layout managers, which this chapter will discuss in later sections.
