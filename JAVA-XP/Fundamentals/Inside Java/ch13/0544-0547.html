Property Sheets

Property sheets are another kind of Dialog in which the properties of a given application element (for example, a cell in a spreadsheet, or a selection of text) are displayed and manipulated. Property sheets are becoming commonplace in applications. See figure 13.8 for an example of a property sheet.


Figure 13.8 A sample property sheet.

Property sheets generally consist of the following:

  • GUI components that display and enable the user to change properties. In figure 13.8, these components include TextField, CheckBox, Choice, and List.

Page 545
  • Notebook-style tabs. If too many properties exist, they sometimes are organized with the use of "tabbed components," which are components that resemble notebook tabs. When the user clicks on a tab, the information on that page is displayed.

NOTE
Please note that the Java AWT does not have the tabbed components, but that they are available from Java component vendors.

  • Two or more buttons. Property sheets almost always have an OK button and a Cancel button. Some sheets may have an Apply button for applying changes without closing the dialog box, and a Reset button for resetting the properties to their original state.
  • A preview area in which a visual indication of the property changes can be seen before the user commits to the selection. The example in figure 13.8 has a preview area that illustrates the effects on the font changes. A Canvas component is well suited for the preview area.

Message boxes and property sheets are not supplied with the JDK, but as you have seen, you can certainly build them for use within your applications by using the Dialog class with other AWT components.

FileDialog and Other Common Dialog Boxes

Certain dialog boxes are implemented in different ways depending on the platform, but have a common function. These dialog boxes have a common look, however, among applications on the same platform. The FileDialog is an example of such a dialog box. You can use the FileDialog class within your application to enable the user to choose either a file to load or a file in which to save their information. Figure 13.9 illustrates the FileDialog prompting the user for a file to load.


Figure 13.9 Example of a common FileDialog.

Page 546

The following code fragment is used to create the FileDialog in figure 13.9: 

FileDialog fd = new FileDialog(this, "Open", FileDialog.LOAD);
fd.setDirectory("c:\\java");
fd.show();

The first line creates a FileDialog, named fd, and passes it three parameters:

  • A frame that owns the FileDialog
  • A String to be displayed on the title bar of the FileDialog
  • An indication of whether this is a load FileDialog or a save FileDialog

General User Input Dialog Boxes

Some dialog boxes are simply used in an application when the application requires input from the user. These dialog boxes have very general uses and have no typical case. Developers are on their own when designing them, but some good GUI style guides on the market can help. A good source for design ideas is to examine the design of dialog boxes in world-class applications.
