ScrollPane constructor takes an int as a parameter, which can be any one of the following final static variables:

  • SCROLLBARS_ALWAYS—This variable places horizontal and vertical scrollbars in the ScrollPane, regardless of whether the child component is larger than the viewport.
  • SCROLLBARS_AS_NEEDED—This variable places a scrollbar in the ScrollPane only when a dimension of the child component is larger than the viewport. If, for example, the dimensions of the viewport are 200 pixels wide by 300 pixels high and the child component is 150 pixels wide by 600 pixels high, then only the vertical scrollbar would appear in the ScrollPane.
  • SCROLLBARS_NEVER—Scrollbars will not appear in the ScrollPane. In this case, the only way to scroll is from the program, by using the setScrollPosition() method.

In the NoScrollPain example, scrollbars will only appear when needed—which is always, because both dimensions of the child component are larger than the corresponding viewport dimensions.

Next, in lines 26_32, the unit increment and block increment values are set for the horizontal and vertical scrollbars. Because the scrollbars are encapsulated within the ScrollPane, methods of the ScrollPane class must be called to obtain a reference to them. The getHAdjustable() method returns a reference to the horizontal scrollbar, and the getVAdjustable() method returns a reference to the vertical scrollbar. As you recall from previous sections in this chapter, scrollbars implement the adjustable interface—hence the getHAdjustable() and getVAdjustable() method names.

The containers discussed thus far are displayed within other containers. The rest of the containers appear in their own window, and consequently are inherited from the Window class (see fig. 13.1). The Dialog class is the first class to be explored.

Dialog Class

Dialogs, commonly referred to as dialog boxes, generally are used to display and gather information from the user. Dialog boxes can be used in many different ways:

  • Message boxes
  • Property sheets
  • General user input dialog boxes
  • FileDialog, and other common dialog boxes

The following sections discuss each one in more detail.

Message Boxes

Message boxes are special purpose dialog boxes that give the user some information and then wait until the user responds to the information. When, for example, you try to exit a program without saving your work, a message box may ask, "Exit without saving?". Three buttons appear on the message box with choices such as "Yes, "No," and "Cancel." Message boxes usually are modal, which means that you must respond to them before you can interact with the rest of the application.

The following code fragment builds the user interface of the message box described

Dialog msgDialog = new Dialog(this, "My Application", true);
Panel messagePanel = new Panel();
messagePanel.add(new Label("Exit without saving?"));
msgDialog.add("Center", messagePanel);
Panel buttonPanel = new Panel();
buttonPanel.add(new Button("Yes"));
buttonPanel.add(new Button("No"));
buttonPanel.add(new Button("Cancel"));
msgDialog.add("South", buttonPanel);
msgDialog.setSize(300, 150);
msgDialog.show();

The following steps detail the creation of this message box.

  1. Create the Dialog: 
          Dialog msgDialog = new Dialog(this, "My Application", true);

    This first line creates the Dialog, passing it three parameters:

    • The Frame that owns the dialog box. All dialog boxes are owned by a Frame, which is usually the main Frame of the application.
    • A String for the title bar of the dialog box. Frequently, developers put the name of the application in the title bar, so that when the message box appears, the user knows from which application it came.
    • The desired modality. If true, the dialog box blocks the user from
    • interacting with the rest of the application until the user has dealt with it.
  1. Create a Panel in which to put the "Exit without saving?" message, add the message to the Panel, and then add the Panel to the "Center" area of the Dialog: 
Panel messagePanel = new Panel();
messagePanel.add(new Label("Exit without saving?"));
msgDialog.add("Center", messagePanel);

The default layout manager for a Dialog is the BorderLayout, which will be covered later in this chapter.
