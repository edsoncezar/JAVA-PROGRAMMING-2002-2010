As you have seen, the List component displays a list of choices to the user. Another component, called the Choice component, also presents a set of choices, although in a slightly different manner, as the following section details.

Choice Components

A Choice—commonly referred to as a drop-down list—is similar to a List, except that the Choice component is displayed as only one line of text. When the user clicks this component, the list of choices drops down. Unlike a List, a Choice can accept only one selection at a time. The following example creates a Choice with the days of the week: 

Choice days = new Choice();
days.add("Sunday");
days.add("Monday");
days.add("Tuesday");
days.add("Wednesday");
days.add("Thursday");
days.add("Friday");
days.add("Saturday");
add(days);

TextField and TextArea Components

The TextField and TextArea classes are used to display and receive text input from the user. The main difference between the two is that a TextField has one line of text, whereas TextArea can have multiple lines. As shown earlier in this chapter in figure 13.1, both classes are derived from the TextComponent class and share many of the same methods. The following example creates a TextField that is 20 columns wide and contains the words "Change me!". This example also creates an empty TextArea that is 20 columns wide and four rows high. 

TextField tf = new TextField("Change me!", 20);
TextArea ta = new TextArea(null, 4, 20);
add(tf);
add(ta);

Because the TextField and TextArea classes are derived from the TextComponent class, some methods used with the TextField and TextArea components are defined there. The ScrollBar Component

Scrollbars usually are used in conjunction with an area to be scrolled. Sometimes, however, a scrollbar can be used as a "slider" to enable the user to input a value. As

Figure 13.4 Scrollbar elements and properties.

figure 13.4 illustrates, scrollbars can be horizontal or vertical. This figure also shows some properties associated with scrollbars.

A scrollbar has the following properties:

  • Orientation: A scrollbar can be either horizontal or vertical.
  • Maximum: This property represents the highest value that the scrollbar can represent.
  • Minimum: This property represents the lowest value that the scrollbar can represent.
  • Value: This property is the current value represented by the scrollbar.
  • Visible Amount: When used in conjunction with an area to be scrolled, the size of the bubble in the scrollbar should be proportional to the visible area of the document being scrolled.
  • Unit Increment: This property refers to the amount that the value will be increased when the user clicks the right or top arrow.
  • Block Increment: This property refers to the amount that the value will be increased when the user clicks the area between the bubble and the right or top arrow.

You can get and set these properties by using the get and set methods associated with each property. Use the setVisibleAmount() method, for example, to set the visible amount. The adjustable interface, which the Scrollbar class implements, defines all these methods.
