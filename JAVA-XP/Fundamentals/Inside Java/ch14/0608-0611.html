AdjustmentEvent Class

The AdjustmentEvent class contains the event type ADJUSTMENT_VALUE_CHANGED, which is the second semantic event type discussed earlier. An AdjustmentEvent can be fired by:

  • Manipulating a scrollbar.
  • Manipulating a custom component, such as a slider. As you'll see in Chapter 15, "Java Beans: Java's Component Architecture," custom components (Beans) can fire any of the events in the AWT.

For an example of using the AdjustmentEvent class, see the ScrollbarSlider example in Chapter 13. This example uses the getValue() method of the AdjustmentEvent class to return the current value of the scroll bar.

The getAdjustable() method returns a reference to the component that fired the AdjustmentEvent. This reference is of type Adjustable, because all components that fire this event must implement the Adjustable interface. Note that this method returns the same reference as getSource() does, except that it is of type Adjustable, which means that you don't have to cast it. (Chapter 13, "The New, Improved AWT: GUI Components, Containers, and Layout Managers," discusses the Adjustable interface.)

If you need to know what type of adjustment caused the AdjustmentEvent to fire, call

ItemEvent Class

The ItemEvent class contains the event type ITEM_STATE_CHANGED, which is the third semantic event type discussed earlier. An ItemEvent can be fired by selecting or deselecting an item in a

  • List
  • Choice
  • Checkbox
  • CheckboxMenuItem
  • Custom component (Java Bean)

To determine which type of state change caused the ItemEvent to fire, you can call the getStateChange() method. This returns one of the following values defined in the ItemEvent class, which indicates whether the user selected or deselected the item:

  • SELECTED
  • DESELECTED

ComponentEvent Class

The ComponentEvent class is the root class for all low-level event classes. This class introduces a method named getComponent(), which can be used on any of its subclasses to return a Component reference to the component that fired the event. This returns the same reference as the getSource() method of the EventObject class, except that it is a Component reference, not an Object reference.

You can use the inherited getId() method to determine which event type was fired. These event types correspond to the ComponentListener methods, so it is not necessary to check for event type when you are in one of these methods. The getId() method is useful, however, when using the Extended Component Event Model, which is discussed later in this chapter. The ComponentEvent event types are:

  • COMPONENT_MOVED
  • COMPONENT_RESIZED
  • COMPONENT_SHOWN

FocusEvent Class

The FocusEvent class contains the following event types, which correspond to the methods in the FocusListener class:

  • FOCUS_GAINED
  • FOCUS_LOST

As with any other subclass of AWTEvent, you can use getId() to determine which of these event types was fired.

Any component in the Component hierarchy can receive a FocusEvent. For more information on the FocusEvent class, refer to the following Web page:
