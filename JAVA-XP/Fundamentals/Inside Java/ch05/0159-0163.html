Arrays of Objects

An array is an object within Java. Each element of the array can be a Java basic type or a classname. In the latter case, arrays can be used to store Java objects. The array then becomes a Java object that contains other Java objects.

All array objects have a variable length associated with them that can be used to determine the size (also called length) of the array. To get the length of the array, you can use the following syntax: 

arrayname.length;

Consider the following sample program: 

int[] xi = new int[50];
char[] xc = {`E', `L', ` `, `M', `O', `R', `Y', `A'};
float[] xf = {_3.0F, 2.0F, 9.7F, 9.02F, 344.23F};
short[] xs;

int xsLength;

xsLength = xc.length + xf.length;
xs = new short[xsLength];

System.out.println("xi length = "+xi.length);
System.out.println("xc length = "+xc.length);
System.out.println("xf length = "+xf.length);
System.out.println("xs length = "+xs.length);
The output of the preceding code fragment would be:
xi length = 50
xc length = 8
xf length = 5
xs length = 13

Consider another example, where you need to create an array of nine objects. Assume that the object is defined by the class Aircraft that has been defined elsewhere, and that this class has a single constructor with no arguments Aircraft(). 

Aircraft[] flights = new Aircraft[9];

The preceding statement defines the array flights of nine elements. The array object is instantiated, and each element in the array is the type of the class Aircraft. At this point each element of the array is ready to hold an Aircraft object, but these objects must be instantiated in a separate step by calling the constructor for the class Aircraft. The

following illustrates how this can be done: 

for (int x = 0; x < 9; x++)
{
flights[x] = new Aircraft();

} // end for

This loop will instantiate each of the elements of the array and assign it to a cell in the array flights.

In preceding examples of array definitions where the element type of the array is a Java basic type, there was no need to call a constructor for each element. This is because Java basic types are instantiated by declaring them; that is, space for Java data types need not be defined in a separate step. Java objects behave differently from Java basic data types. For Java objects a constructor must be called to instantiate the object.

Defining Multidimensional Arrays

So far, all the examples of arrays use single-dimensional arrays. Many problems may require multidimensional arrays, such as matrix representation or representation of a simple spreadsheet. Another example would be if you wanted to store the best timings for a given number of athletes of swimmers for several competitive events. The idea of a two-dimensional array is shown in figure 5.18. There are two indexes, one that indexes the columns, and the other that indexes the rows. For the cell that has a row index of x, and a column index of y, the cell is referenced by 

    mat[x][y]


Figure 5.19 Array of array objects.

If this matrix has 10 rows and 12 columns, it is defined by using either of the following syntax: 

int mat[][] = new int[10][12];

or 

int[][] mat = new int[10][12];

The actual array that is created is an array of arrays as shown in figure 5.19. Each of the elements of the array mat[0] to mat[9] contains another array object that has 12 elements of

type int.

You can extend this idea to an array of any dimensions. An array of four dimensions, for example, could be defined as either of the following: 

int matx[][][][] = new int[10][20][30][40];

or 

int[][][][] matx = new int[10][20][30][40];

You can even define variable size arrays of arrays as long as the first dimension of the array has been specified beforehand. Consider the following example: 

int xa[][] = new int[6][];

These secondary arrays are instantiated in the separate assignment statements. Figure 5.20 shows a graphical representation of the variable sized array. You must be careful how you index this array. If you exceed the valid index range, an exception will be raised.
