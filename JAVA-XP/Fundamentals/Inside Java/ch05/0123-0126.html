The System.out.println() method was used often in the previous chapters. You are in a better position to understand this method now that the use of objects has been formally introduced. Java has a class called System, defined in the package java.lang, that has only static variables and methods. Two of the very important static variables are in and out. The variables in and out define objects of the class InputStream and PrintStream, respectively. Because out is an instance of the class PrintStream, all the methods that are available for PrintStream can be invoked by using this notation: 

Object.method();

In this example, Object is the object System.out and method is println().

Lines 21_23 contain the definition of the objects date1, date2, and date3. Notice that in these definitions, the objects are initialized to a null value. On line 25, the value of the date1 object is printed out. From figure 5.2, you can see that the value printed out is the string "null": 

Printing undefined date value null

All the predefined Java classes are written with the consideration that if an attempt is made to access an undefined value, a value of null is set for the object.
