Thus, the first definition leads to two instantiations, one of which is the instantiation of the temporary String object. Because the temporary String object has the same value as the name1 object, this is a little bit of a waste. For this reason, the second definition is preferred for string constants.

You can also use an array of characters or an array of bytes to instantiate a String object: 

char[]     xc = {`K', `R', `I', `S', `H', `N', `A'};
String vishnu = new String(xc); // vishnu = "KRISHNA"

byte[] ba = {`1', `A', `C'};
String bs = new String(ba, 0, 1, 2); // bs = "AC"

After a String object has been created, it is treated as a constant, and you should not attempt to modify its value. If you want to define a String variable whose value can be modified, you must use the StringBuffer class.
