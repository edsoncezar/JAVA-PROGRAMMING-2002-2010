So far you have learned about basic Java data types and control flow statements. Besides basic data types, Java contains many predefined classes. As you learned in Chapter 2, "Getting Started with Java," when you create an instance of a class, that instance is called an object. In this chapter, you will learn how to use those objects. Note that you have not yet learned to define a full blown class type, although you have learned to define a simple class to encapsulate the main() method. It is not necessary to know the full details of how a class is defined to use classes.

Sometimes one needs to access aggregates of a variable of the same type through some kind of indexing mechanism. Java provides the array mechanism to support indexing of aggregates of a data type. The use of arrays has become fundamental to modern programming languages. Any data structure that consists of cells of data is a good candidate for representation by arrays. The mathematical concept of matrix can be easily represented as an array.

This chapter also discusses how arrays are defined in Java. You learn how to define multidimensional arrays, how to initialize them properly, and how arrays are different from ordinary variables. In many languages the concept of strings of characters is represented as an array of characters. In Java the string data type is implemented by the class String. Using Java Objects

The types of Java data types that you examined in Chapter 3, "Experimenting with Java Data Types and Variables," have been simple "unadorned" data types such as integers, floating point data, Booleans, and characters. These basic data types are available in most languages. One of the things that distinguishes Java as a programming language is its object-oriented nature that enables the use of a higher level representation of data called an object. An object is a collection of data items along with the methods (programs or functions) that operate upon the data.

Creating Objects

For an object, the format of the data and the definitions of the methods that operate upon the data are described by a class. A class, in Java, is a template for how the object is to be constructed. Creating an object from the data template represented by the class is called creating an instance of the class. The object instance is assigned a name that is similar to the names used for variables of the basic Java data types. This name is called the object name.

The object name can be of any length. Some limitations exist on the characters you can use to create the object name. The object name, like a variable name in Java, must begin with a letter, followed by any combination of letters and digits. A letter is a set consisting of letters a_z, A_Z and the underscore (_) and dollar ($) characters.

Recall that if you needed to create a variable of a basic Java data type, you could use the type name followed by the variable names as shown in the following example: 

int     aVarName;
float     area;
char     inpCh;

If you need to create an object instance, the syntax is similar to creating a variable name. You, however, must also use a constructor for that object. What is a constructor? A

constructor is a special method that contains the program code to build the object. Building the object may involve initializing the data elements in the object. All classes must have this constructor method. If one has not been defined, the Java language supplies this default constructor. The name of the constructor method is the same as the name of the class. Consider a class called Aircraft: 

class Aircraft
{
// Data elements describing the class

// Constructor method
Aircraft() {…}
}

Notice that the following line in the preceding class definition is the constructor method: 

Aircraft() {…}

Notice that the name of the constructor method is the same as the class name—this is what is used to identify the constructor method.

The { … } contains the statements that implement the constructor method. The parentheses after Aircraft and before the { … } contain the arguments to the method. Arguments are data that is passed on to the method.

To create an object of this class—called instantiating the class—you can use the following syntax: 

Aircraft     boeing_747 = new Aircraft();

The object name is boeing_747. The = new Aircraft() initializes the object with the object created by the constructor. Notice that the call to the Aircraft() constructor is preceded with new. Whenever you call a constructor to create an object you must precede it with the new keyword.

The object boeing_747 could also have been created in two steps, as shown next: 

Aircraft     boeing_747;
boeing_747 = new Aircraft();

The statement Aircraft boeing_747; just declares the variable; it does not define the variable. No object is associated with this variable; it is a reference to nothing. More specifically, the variable is set to the value of null. The reserved word null is used to indicate that no object is associated with the boeing_747. The second statement boeing_747 = new Aircraft(); defines the value of the object.
