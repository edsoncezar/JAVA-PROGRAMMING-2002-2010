The classtype is the name of the type wrapper classes such as Boolean, Character, Double, Float, Integer, Long; the basictype is the name of the corresponding basic type such as Boolean, character, double, float, int, long.

The first method type "public classtype(basictype)" is used as a constructor to build an object of the classtype. The following are examples of how this can be used: 

Integer ic = new Integer(99);
Boolean btrue = new Boolean(true);
Character cc = new Character(`K');
Float     fc = new Float(3.14159F);
Double dc = new Double(1.414);

After the objects of the type wrapper class have been created, you can use the methods for that class on that object.

The second method type "public basictype classtype Value()" is used to extract the native Java basic data type value from the type wrapper class. Before performing any arithmetic or Boolean operations, you must extract the value of these basic types. The examples below extract the basic type values from each of the type wrapper classes defined in the previous example: 

int iv = ic.intValue(); // Set to 99
boolean bv = btrue.booleanValue(); // Set to true
char cv = cc.charValue(); // Set to `K'
float fv = fc.floatValue(); // Set to 3.14159F
double dv = dc.doubleValue();     // Set to 1.414

The third method type "public String toString()" is used to convert the value stored in the type wrapper class to an equivalent string representation. This method is primarily used for creating string representations of the Java data types for display or storing them into a text file. The examples below can be used to display the data types on the console. 

System.out.println(ic.toString());
System.out.println(btrue.toString());
System.out.println(cc.toString());

The fourth method type "public boolean equals(Object obj)" is used to test if the value in one type wrapper object is the same as the one in another type wrapper object. This method is a convenient way to check the equality of values contained in the wrapper objects. Note that you cannot use the equals operator (==) to compare two class objects. The fifth common method type "public int hashCode()" is used to return an internal integer value that contains a value that is a mathematical summation of the value that is encapsulated in the type wrapper class. The hash code is used for operations with the Hashtable class in package java.util.

The Boolean Type Wrapper Class

The Boolean type wrapper, because it encapsulates either a true or false value, does not implement any other methods besides the common methods discussed in the previous

Page 140

section. The Boolean class has two public static variables defined as follows within that class: 

public static final boolean TRUE = new boolean(true);
public static final boolean FALSE = new boolean(false);

These statements define TRUE and FALSE as Boolean objects, containing the Boolean value TRUE and FALSE, respectively. The use of the reserved words public, static, and final as qualifiers for these types has an interesting meaning.

The use of the reserved word public before a method declaration or class variable definition, makes the method or variable accessible to programs outside that class. In this case, the objects TRUE and FALSE are available for use outside the Boolean class.

The use of the reserved word static for a class variable makes the variable available to all instances of that class. In other words, all object instances will share the same copy of the static object. Because the variable is not especially associated with any particular object instance, you can access the variable without defining an instance of this class. You could, for example, access these Boolean objects by using the following notation: 

boolean.TRUE
boolean.FALSE

Notice that Boolean is the name of the class and not the name of an object instance of that class.

The use of the reserved word final for a class variable makes the variable constant. In other words, the value of the constant variable cannot be modified. Therefore, any attempts to modify the values of the Boolean objects TRUE and FALSE will be caught by the compiler and the runtime Java environment.

The Character Type Wrapper Class

Besides the common methods for type wrapper classes, the Character class provides additional methods that are all static. Because these methods are all static, they can be invoked without creating an object instance of the Character class, using the following syntax:
