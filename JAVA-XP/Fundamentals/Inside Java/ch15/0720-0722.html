Using the Jar Tool to Package the Scope Bean

The steps for packaging a bean in a JAR file are as follows:

  1. Create a file that supplies additional information to the manifest file.

    The additional information to be supplied is that the Scope.class file is a Java Bean. We'll arbitrarily name this file manifest.tmp, which contains the following lines: 
    Manifest-Version: 1.0
Name: chapter15/code/scopejar/Scope.class
Java-Bean: True
  2. Run the jar tool to create the JAR file.

    Assuming that the default directory is at the base of the chapter15/code/scopejar path, and that the Scope.class and Scope.ser files are the only files in the chapter15/code/scopejar directory, execute the following command: 
    jar cfm scope.jar manifest.tmp chapter15/code/scopejar/*

    This creates a JAR file named scope.jar by using manifest information from manifest.tmp and containing the files in the chapter15/code/scopejar directory.

This JAR file can now be used directly by a builder tool, or the files can be extracted and

Summary

This chapter focused on the new Java component architecture known as Java Beans. It defined Java Beans and discussed the most important features: properties, events, and methods. Several examples demonstrated how you can build your own Beans and emit the desired events. Advanced topics, such as bound properties and introspection, were covered as well.

The next chapter covers the design goals and capabilities of Java Remote Method Invocation (RMI) and how to develop Java programs that use RMI. This chapter also discusses Object Serialization, how serialization enables RMI, and how serialization enables persistent Java objects.
