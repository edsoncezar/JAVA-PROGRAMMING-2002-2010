Drawing Lines

The main method used to draw a line is the method drawLine() (see lines 114_123 in listing 9.1). This method takes parameters for the start of the line, and the end of the line. If, for example, you wanted to draw lines diagonally inside an applet region and the size is 200 pixels by 200 pixels (see fig. 9.2), you would use code similar to the following: 

public void paint(Graphics g)
{
      g.drawLine(0, 0, 200, 200);
      g.drawLine(0, 200, 200, 0);

Drawing Rectangles

The Graphics class has methods to draw rectangles with square corners, rectangles with rounded corners, and three-dimensional rectangles with shaded borders. For each of these rectangles, you have a choice of two methods:

  • The method that has the "draw" prefix draws an outline form of the rectangle.
  • The method that has the "fill" prefix draws the rectangle filled with color. The color value can be set by using the setColor() method.

Consider, for example, that you want to draw two rectangles: one filled and one that is in an outline form (see fig. 9.3). The inner rectangle is filled, so you will use the fillRect() method (see lines 126_128 of listing 9.1). The outer rectangle is in outline form, so you will use the drawRect() method (see lines 131_136 of listing 9.1). The arguments to these methods are the <x,y> coordinate of the top-left-hand corner, the width of the rectangle, and the height of the rectangle.

Drawing Rounded Rectangles and Circles

The Graphics class has methods to draw rectangles with rounded corners. A circle is seen as an extreme case of a rounded rectangle.

Consider, for example, that you want to draw four rectangles: two that are filled with a color and two that are in an outline form (see fig. 9.4). To create the filled, rounded rectangles, you will use the fillRoundRect() method (see lines 159_166 of listing 9.1); to create the outline rounded rectangles you will use the drawRoundRect() method (see lines 149_156 of listing 9.1).

Figure 9.4 Example of rounded rectangles.

The arguments to these methods are the <x,y> coordinate of the top-left-hand corner, the width of the rectangle and the height of the rectangle, and two additional arguments. These two additional arguments define how far along the edges of the rectangle that the arc for the corner will start. The first of the two additional arguments is the arc width, and this is the diameter of the circle that will produce the horizontal arc (see fig. 9.5); the last argument is the arc height, and it is the diameter of the circle that will produce the vertical arc.
