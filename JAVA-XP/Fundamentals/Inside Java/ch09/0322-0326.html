Drawing 3D Rectangles

You can use the Graphics class to simulate three-dimensional rectangles. These rectangles produce a shadow effect that make them appear three-dimensional.

What if, for example, you want to draw two rectangles: one that is filled with a color and looks like a raised 3D button and the other that is in an outline form but appears pressed. To create these 3D rectangles, you will use the fill3DRect() method (see lines 176_181 of listing 9.1) and the draw3DRect() method (see lines 169_173 of listing 9.1).

The arguments to these methods are the <x,y> coordinate of the top-left-hand corner, the width of the rectangle and the height of the rectangle, and a Boolean argument. If the Boolean argument is true, the rectangle is raised; otherwise, the rectangle is indented.

The following is the paint() method that produces a solid-raised rectangle and an outline-indented rectangle. 

public void paint(Graphics g)
{
         g.fill3DRect(10, 10, 20, 20, true);
         g.draw3DRect(10, 50, 30, 15, false);

} // end paint

Drawing Ovals

You can use the Graphics class to draw ellipses and circles. You can also produce the same effect by using rounded rectangles, except that the use of the oval method is easier to use to produce ellipses and circles.

Consider, for example, that you want to draw two ovals, one that is a circle filled with a color, and the other that is an ellipse in an outline form (see fig. 9.6). To create these oval shapes, you will use the fillOval() method (see lines 190_193 of listing 9.1) and the drawOval() method (see lines 184_187 of listing 9.1).


Figure 9.6 Drawing ovals.

The arguments to these methods are the <x,y> coordinate of the top-left-hand corner, the width of the oval, and the height of the oval. The following is the paint() method to produce the shapes in figure 9.6: 

public void paint(Graphics g)
{
      g.fillOval(10, 10, 20, 20);
      g.drawOval(50, 10, 20, 40);

} // end paint

Drawing Arcs

Arcs are segments of the circumference of an oval. To define an arc, you must first see it as part of an oval (see fig. 9.7). The arc is only a portion of the oval.

Page 324

The arc drawing methods are drawArc() (see lines 196_206 in listing 9.1) and fillArc() (see lines 209_216 in listing 9.1). The first four parameters for drawing an arc are the same as that of an imaginary oval of which the arc is part. The fifth parameter is the start angle of the arc. The start angle is measured with respect to the center of the circle. Zero degrees is at the 3 o'clock line; 90 degrees is at the 12 o'clock line; 180 degrees is at the 9 o'clock line; and 270 degrees is at the 6 o'clock line (see fig. 9.8). The sixth, and last parameter, is the arc angle, and it is the number of degrees that you must "sweep" from the start angle to create the arc. A positive value of the arc angle will result in a counter-clockwise motion, and a negative value will result in a clockwise motion.

Drawing Polygons

Polygons are arbitrary shapes with many sides drawn by straight lines. Familiar figures, such as squares, rectangles, triangles, pentagons, and so on, are all examples of closed polygons. Polygons are closed because they bound a region of two-dimensional space.

Polygons are drawn by the methods drawPolygon() (see lines 219_230 in listing 9.1) and fillPolygon() (see lines 316_330 in listing 9.1). These methods accept two types of arguments—arrays of x- and y-coordinates or an instance of the Polygon class.

If you are using arrays, you must provide three arguments. The first argument is an array of x-coordinates; the second argument is an array of y-coordinates; the last argument is the number of elements in each of the two array arguments. The number of elements in each of the x- and y-coordinate array must be the same.

The polygon draw methods do not automatically close the shape. If you want a closed shape, the last point in the array must be the same as the start point in the array. If you draw a filled polygon, the method will join the start and end points.
