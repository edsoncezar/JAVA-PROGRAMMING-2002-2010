If you need to clear the entire applet, first get the dimensions of the applet. You can get the dimensions by invoking the size() for the instance of the applet by using this.size(). The size() method is inherited by your applet from the Component class, and it returns an instance of the class Dimension. The Component class is one of the superclasses in the inheritance hierarchy chain for the Applet class.

The Dimension object is returned by a number of other methods in the java.awt package, and it is a convenient way of specifying dimensions such as a rectangular area as an object. The top-level specification of Dimension is included here to assist in understanding its behavior: 

public class Dimension extends Object
{
      // Public constructors
      public Dimension();
      public Dimension(Dimension d);
      public Dimension(int width, int height);

      // Public instance variables
      public int height;
      public int width;

      // Public instance methods
      public String toString(); // Overrides Object.toString();

} // end Dimension

Therefore, to get the width and height of a class, you could use the following: 

this.size().width                    // Gets the width

this.size().height                    // Gets the height

In the preceding statements this.size() returns a Dimension object, so you can access the public instance variables width and height of this object. The Dimension object is a temporary object and is created each time this.size() is called, which is a little inefficient. The following paint() method shows the applet area being cleared and filled with the background color, using only a single instance of the Dimension class: 

public void paint(Graphics g)
{
      Dimension d = this.size();

      g.clearRect(0, 0, d.width, d.height);

} // end paint

Setting Background and Foreground Colors

When you clear a rectangle by using the clearRect() method, the cleared area is filled with the background color. By default, this color is a dark gray. If you want to use a different background color, you must use the setBackground() method. There is also a corresponding setForeground() method that sets the foreground color. Both of these methods are inherited by your applet from the Component class. They both take a single argument that is of instance Color. To change the background of the applet to a yellow color, for example, you would use the following: 

setBackground(Color.yellow);

When you use setForeground(), you change the color of all text or graphic elements that have been drawn in a single stroke. To change the color of all graphic elements that have been drawn to pink without redrawing the applet, for example, you would use the following: 

setForeground(Color.pink);

You have already seen the use of the setColor() method defined in the Graphics class (see lines 46_50 of listing 9.1) to set the color for all future drawing operations. You can also use the getColor method defined in the Graphics class (see lines 42_43 of listing 9.1) to obtain the current color setting. To change the foreground color of the applet to the current color setting, for example, you could use the following:
