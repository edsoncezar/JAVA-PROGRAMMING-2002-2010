CHAPTER 4

Java Flow Control

In the previous chapter, you learned about the different Java basic data types. You may have noticed that in all the examples in the previous chapters, only the assignment statement was discussed. Furthermore, in these examples, the statements were executed sequentially one after another. Programs that are used to model real-life systems are seldom that simple. You are often required to execute statements depending on a test condition.

Java provides many statements to affect the flow of logic within a program, so that depending upon a Boolean condition that is evaluated, a different part of the program is executed. In this chapter you learn to control the sequence of statements that are executed in a program. These statements are generically referred to as control flow statements and include statements such as the if-statement, while-statement, do-while statement, switch statement, and so on.

Assignment Statement

You have seen mostly examples of assignment statements in previous chapters. Assignment statements have the following general syntax: 

L-value = expression { = expression };

The L-value is any program item to which a value can be assigned. The most common examples of this are variables such as those discussed in Chapter 3, "Experimenting with Java Data Types and Variables." The expression could be an arithmetic expression or a Boolean expression consisting of Boolean operators and relational operators
( < > == != <= >=). The value of expression is assigned to the L-value. The type of the expression must be consistent with the type of the L-value. You, for example, cannot assign a Boolean expression to an int type variable. Among integers and floating point

Page 77

types, you can perform an assignment with the general rule that the precision and range of the expression should not exceed that of the L-value. Between integer and floating point data types, you can force a conversion by using a type cast. You can, for example, force a conversion from a double floating point expression to a long type, but it will result in a loss of precision. Consider the following example: 

long          circumference;
double     PI = 3.1415926536;
double      radius = 33.0;
circumference = (long) 2.0 * PI * radius;

The use of the cast specifier (long) for the double floating point expression converts it into a long type, but any fractional part in the numeric value is lost.

The use of the braces { } in the syntax of the assignment statement indicates that the contents within braces can occur zero or more times. The { } characters serve as meta-characters to describe the syntax of the assignment statement and are not part of the assignment statement. This form of assignment statement is convenient because it can be used to set several variables to the same value, as seen in the following fairly common example where several variables need to be set to the same value: 

short  x, y, z;

x = y = z =  0;

Another form of assignment statement combines an arithmetic operator, shift, or bitwise operator with the assignment operator. The general syntax of this assignment operator is the following: 

L-value op= expression;

The L-value and expression have the same meaning as discussed before, and op is any of the following operators:

+ _ * / % >> << <<< & ^ |

The above assignment statement is equivalent to the following:

L-value = (L-value) op (expression);

Notice the parentheses around the expressions. Therefore, the following statement

a *= b_c;

is actually the following:

Page 78

a = a * (b_c);

rather than

a = a*b_c;

Essentially, this type of statement is a shortcut expression. Here are some examples of this statement: 

int     x = 1;
int      y = 7;

x += 1;     // x is now 2
y _= 3;     // y is now 4
x <<= 1;     // x is now 4
y >>= 1;     // y is now 2
y *= x + 1;     // y is now 10
y /= x _2;     // y is now 5
y %= 3;     // y is now 2
y ^= 3;     // y is now 1

x &= 2;     // x is now 0
y |= 3;     // y is now 3

Assignment operators of the type just described can be quite intuitive to understand. When, for example, you want to increase x by 4, you do not say "take x add 4, and then assign it to x." Instead, it is more natural to say "add 4 to x." This is exactly what is meant by the expression x += 4.

NOTE
The L-value op= expression expression preserves compatibility with C and C++. Originally, this existed in the C language as an aid for compilers to perform optimization. In the case of +=, for example, the compilers could generate code that would evaluate the right-hand side and add it to the L-value.

Expression Statement

An expression statement is any expression that is on a line by itself and ends with a semicolon. In truth, the assignment statement is also an expression statement because the following construct is an expression:

Page 79

L-value = expression

Therefore, any of the following are expressions: 

x = 0
y = 3

Notice, that these expressions do not end in a semicolon. If they did, they would become an expression statement—also called an assignment statement, in this case, because of the use of the assignment (=) operator.

An expression statement is the expression followed by a statement. Consider the following: 

long     z = 11;

++z;

This expression statement has the result of incrementing the value of z by 1. It is equivalent to each of the following expression statements: 

z += 1;
z = z + 1;
z++;

Similarly, the decrement operator can be used as an expression operator. Thus, the following statements are all equivalent: 

--z;
z--;
z _= 1;
z = z_1;

Each of these expressions results in a change in value. If, however, you write the following statement in Java, it will not result in any change in value: 

x + 3;

Although this statement compiles correctly, it is a useless statement because it does not cause any change in the program logic. The reason for this is because there is no L-value in the statement.
