In this switch statement, if the value of x is _1, the following statements are executed: 

y++;
z++;

Notice that the statement z++ is also executed because the execution falls through to the next case statement due to the absence of the break statement.

If the value of x is 2, the following statement is executed: 

z++;

Notice that execution of further statements for values of x equal to 1 or 2 is terminated when the break statement is encountered.

If the value of x is set to 3, the following statement is executed: 

z _= x;

For all other values of x that are not 1, 2, or 3, the following statement is executed: 

y = _9;

Notice that there is a break statement after the last default statement in the switch, even though it is logically not necessary. This is done in case another case statement is added at the end of the default statement. The break statement prevents an inadvertent fall through to the next case statement.

The while Statement

The while statement is the basic statement that is used for creating a loop. Most programs need to perform some form of repeated action, and the while statement provides a nice solution for performing this repeated action.

The general syntax of the while statement is as follows: 

while (booleanExpression)

      statement

The booleanExpression is evaluated at the beginning of each loop execution. If this expression evaluates to a true, the statement is executed. This statement is any Java statement including the block statement. At the end of the statement execution, control is transferred to the top of the loop where booleanExpression is evaluated again. If the evaluation yields a true result, the statement is executed again. So as long as the

Page 99

booleanExpression evaluates to true, the statement is executed, and this implements a loop. The loop terminates only when booleanExpression evaluates to a false. When the loop terminates, control is transferred to the end of the while loop.

Figure 4.8 Logic flow diagram for a while statement.

Consider the following example of a while statement: 

short    x = 0;
short    max = 40;
double   y = 100000000.0;
double   z = 2.0;

while (x++ < max)
{
      y = Math.sqrt(y);
      z = Math.exp((double)x);

Page 100

 
      System.out.println("y = "+y);
      System.out.println("exp("+x+") = "+z);

} // end while

This while statement will execute until the expression x++ < max becomes false. This will happen when the value of x before the postfix increment is 40. Within the while loop, the value of x will range from 1 to 40. The loop will successively take the square root of a large number 100000000.0 and display the results. Additionally, the loop takes the exponent of values of variable x, and displays the results.

Consider another example of a while statement that is often useful: 

while (true)
{
      // Statements

} // end while

At first sight, this type of while statement may appear strange because you are testing for the constant true in the Boolean expression part of the while statement. Clearly such a test will never evaluate to false, and therefore the loop will continue indefinitely. Actually, this is the precisely the point of this while statement—it sets up an indefinite loop. The indefinite loop is useful for setting up background processes and "server" processes that need to be running all the time. These processes could perform repeated polling action. An example of a server process would be a printer server process, running on a computer with several printers attached to the computer. This print server process would loop indefinitely listening for print requests from other computers.

There is actually even a better way of setting up an indefinite loop. You will learn more about this in the next section on for statements.
