GLOSSÁRIO
J2EE 2 Platform, Enterprise Edition

Este glossário define termos de acordo com seu uso no contexto da JAVA 2 Platform, Enterpresie Edition (Plataforma J2EE). Muitos dos termos possue uso mais genérico. Por exemplo, o termo handle é muito usado nas linguagens de programação modernas, mas aqui ele é definido em termos do seu uso na tecnologia Enterprise JavaBeans(EJB)

A B C D E F G H I J L M N O P Q R S T U V X Z

realm
Uma string, passada como parte de um pedido HTTP, durante a autenticação básica, de define um espaço de um pedido HTTP, durante a autenticação básica, que define um espaço de proteção. Os recursos protegidos em um servidor podem ser particionados em um conjunto de espaços de proteção, cada um com seu próprio esquema de autenticação e/ou banco de dados de autorização.

recurso do sistema de informações da empresa
Uma entidade que oferece alguma funcionalidade de um sistema de informação da empresa aos seus clientes. Alguns exemplos são: um conjunto de registros em um sistema de banco de dados, um objeto comercial em um sistema de planejamento de recursos da empresa e um programa de transação em um sistema de processamento de transação.

Refence Implementation (RI)
Veja "Java 2 SDK, Enterprise Edition (J2EE SDK)", neste glossário.

reino
Veja "domínio de política de segurança", neste glossário.

restrição de autorização
Uma regra de autorização que determina quem pode ter acesso a uma coleção de recursos na Web.

restrição de dados do usuário
Indica como os dados entre um cliente e um contêiner da Web devem ser protegidos. A proteção pode ser a prevenção contra adulteração ou bisbilhotagem sobre os dados.

restrição de segurança
Uma forma declarativa de anotar a proteção desejada do conteúdo da Web. Uma restrição de segurança consiste em uma coleção de recursos da Web, uma restrição de autorização e uma restrição de dados do usuário.

RMI
Remote Method Invocation. Uma tecnologia que permite a um objeto rodando em uma máquina virtual Java invocar métodos em um objeto rodando em outra máquina virtual Java.

RMI-IIOP
Uma versão da RMI implementada para usar o protocolo CORBA IIOP. RMI em cima de IIOP oferece interoperabilidade com objetos CORBA implementados em qualquer linguagem se as interfaces remotas foram definidas originalmente como interfaces RMI.

roolback
O ponto em uma transação onde todas as atualizações de quaisquer recursos envolvidos nessa transação são revertidos.
