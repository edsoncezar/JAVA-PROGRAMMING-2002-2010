GLOSSÁRIO
J2EE 2 Platform, Enterprise Edition

Este glossário define termos de acordo com seu uso no contexto da JAVA 2 Platform, Enterpresie Edition (Plataforma J2EE). Muitos dos termos possue uso mais genérico. Por exemplo, o termo handle é muito usado nas linguagens de programação modernas, mas aqui ele é definido em termos do seu uso na tecnologia Enterprise JavaBeans(EJB)

A B C D E F G H I J L M N O P Q R S T U V X Z

mapeamento de papel
O processo de associar os grupos e/ou diretores reconhecidos pelo contêiner a papéis de segurança especificados do descritor de distribuição. Os papéis de segurança são mapeados pelo Deployer antes que um componente em particular seja instalado no servidor.

mapeamento de servlet
Define uma associação entre um padrão de URL e um servlet. O mapeamento é usado para mapear pedidos aos servlets.

mensagem
No Java Message Service, um pedido, relatório ou evento assíncrono é criado, enviado e consumido por uma aplicação da empresa, e não por um ser humano. Ela contém informações vitais, necessárias para coordenar aplicações da empresa, na forma de dados precisamente formatados, que descrevem ações comerciais específicas.

Message Consumer
Um objeto criado pro uma sessão JMS que é usada para receber mensagens enviadas a um destino.

Message Producer
Um objeto criado por uma sessão JMS, que é usado para enviar mensagens para um destino.

método comercial
Um método de um enterprise bean que implementa a lógica ou as regras comerciais de uma aplicação.

método de criação
Um método definido na interface home e invocado por um cliente para criar um enterprise bean.

método localizador
Um método definido na interface home e invocado pro um cliente para localizar um bean de entidade.

método remove
Método definido na interface local e invocado pro um cliente para destruir um enterprise bean.

método de callback
Métodos de componentes chamados pelo contêiner para notificar o componente quanto a eventos importantes em seu clico de vida.

módulo
Uma unidade de software que consiste em um ou mais componentes J2EE do mesmo tipo de contêiner eum descritor de distribuição desse tipo. Existem três tipos de módulos: módulos de EJB, da Web e do cliente da aplicação. Os módulos podem ser distribuídos como unidades independentes ou montados em uma aplicação.

módulo da Web
Uma unidade que consiste em um ou mais componentes da Web e um descritor de distribuição da Web.

módulo do cliente da aplicação
Uma unidade de software que consiste em uma ou mais classes e um descritor de distribuição do cliente da aplicação.

módulo EJB
Uma unidade de software que consiste em um ou mais enterprise beans e um descritor de distribuição EJB.
