GLOSSÁRIO
J2EE 2 Platform, Enterprise Edition

Este glossário define termos de acordo com seu uso no contexto da JAVA 2 Platform, Enterpresie Edition (Plataforma J2EE). Muitos dos termos possue uso mais genérico. Por exemplo, o termo handle é muito usado nas linguagens de programação modernas, mas aqui ele é definido em termos do seu uso na tecnologia Enterprise JavaBeans(EJB)

A B C D E F G H I J L M N O P Q R S T U V X Z

página JSP
Um documento baseado em texto, usando dados de modelo fixos e elementos JSP, que descreve como processar um pedido para criar uma resposta.

papel (desenvolvimento)
Funções distintas realizadas por um parceiro nas fases de desenvolvimento e distribuição de uma aplicação desenvolvida usando tecnologai J2EE, Os papéis são: Application Componente Provider, Application on Assembler, Deployer, J2EE Product Provider, EJB Contêiner Provider, EJB Server Provider, Web Contêiner Provider, Web Server Provider, Tool Provider e System Administrator.

papel (segurança)
Uma agrupamento lógico abstrato que é definido pelo Application Assembler. Quando uma aplicação é distribuída, os papéis são mapeados para identidades de segurança, como diretores ou grupos, no ambiente operacional.

papel de segurança
Veja "papel(segurança)", neste glossário.

parâmetro de inicialização
Um parâmetro que inicializa o contexto associado a um servlet. Veja "atributo de contexto" e "servlet", neste glossário.

passividade
O processo de transferência de um enterprise bean da memória para o armazenamento secundário. Veja "ativação", neste glossário.

permissão de método
Um regra de autorização que determina que tem permissão para executar um ou mais métodos de enterprise bean.

permissão de segurança
Um mecanismo, definido pela J2SE, usado pela plataforma J2EE para expressar as restrições de programação impostas aos Application Component Providers.

persitência
O protocolo para se transferir o estado de um bean de entidade entre suas variáveis de instância e um banco de dados que está por baixo.

persistência controlada pelo conector
A transferência de dados entre as variáveis do bean de entidade e um gerenciador de recurso controlado pelo contêiner do bean de entidade.

persistência controlada por bean
O controle de bean da entidade para as transferências de dados entre as variáveis do bean da entidade e um gerenciador de recursos.

personificação
Um ato pelo qual uma entidade assume a identidade e os privilégios de outra entidade sem restrições e sem qualquer indicação visível aos destinatários das chamadas do imitador de que ocorreu a delegação. A personificação é um caso de simples delegação.

POA
Portable Object Adapter. Um padrão CORBA para a montagem de aplicações no servidor que são portáveis entre ORBs heterogêneos.

privilégio
Um atributo de segurança que não tem a propriedade de exclusividade e que pode ser compartilhado por muitos diretores.

produto J2EE
Uma implementação de acordo com a especificação da plataforma J2EE

provedor de contêiner da Web
Um fornecedor que oferece um contêiner da Web

provedor de ferramenta
Uma organização ou fornecedor de software que oferece ferramentas usadas para o desenvolvimento, empacotamento e distribuição de aplicações J2EE.

provedor JMS
Um sistema de mensagens que implementa o Java Message Service e também outra funcionalidade administrativa e de controle necessária em um produto de mensagem com todos os recursos.

provedor de servidor da Web
Um fornecedor que oferece um servidor da Web
